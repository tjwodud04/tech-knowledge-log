Breadth-First Search (BFS)

난이도 / Difficulty: Intermediate
날짜 / Date: 2025-12-15

내용

BFS는 그래프의 레벨별(최단 경로, 연결성 검사) 탐색 방법으로, 실제 시스템에서 빠른 도달성 계산과 레벨 기반 분석에 널리 사용됩니다.

작동 원리: BFS는 큐(frontier)를 사용해 시작점으로부터 거리 0,1,2… 순서로 이웃을 방문합니다. 각 정점은 한 번만 방문(visited 집합)하고 간선과 정점 수에 대해 O(V+E) 시간 복잡도를 가집니다. 실무에서는 고도로 병렬화하거나 메모리 최적화(프론티어 압축, CSR 포맷)를 적용합니다.
성능 팁: 대규모 그래프에선 방향 최적화(Top-down vs Bottom-up, Beamer et al., 2012)를 적용해 초고차수 노드 처리 비용을 줄이고, 분산 시스템(Pregel/Giraph)이나 GPU(cuGraph)로 프론티어를 병렬 처리합니다. 메모리 재배치(정점 재넘버링)를 통해 캐시 친화성을 높이는 것이 핵심입니다.

활용

사례: Neo4j에서의 최단 경로(무가중):
코드:
// Neo4j 4.x
MATCH (a:Person {id:'A'}), (b:Person {id:'B'})
MATCH p = shortestPath((a)-[*]-(b))
RETURN p

출처: Neo4j 엔진은 무가중 최단경로에 내부적으로 BFS 변형 사용
사례: GPU 가속(rapids cuGraph)
코드:
# cuGraph 22.12+, CUDA 11.x 환경
import cudf, cugraph
df = cudf.DataFrame({'src':[0,1,1],'dst':[1,2,3]})
G = cugraph.Graph()
G.from_cudf_edgelist(df,'src','dst')
res = cugraph.bfs(G, start=0)

출처: RAPIDS cuGraph (NVIDIA)
실무 패턴:
- 작은 지름 그래프는 BFS가 빠름(소셜 네트워크 탐색)
- 초대형 그래프는 방향 최적화 + 재넘버링 + 분산/가속기 병행
참조: Beamer et al., "Direction-Optimizing Breadth-First Search", 2012; Graph500 구현 채택

Content

BFS explores a graph level-by-level (useful for unweighted shortest paths and reachability) and is widely used in production for fast neighborhood queries and graph analytics.

How it works: BFS uses a queue (the frontier) to visit nodes in increasing distance order from a source. Each vertex is marked visited to avoid repeats; complexity is O(V+E). In production, BFS is optimized by storing graphs in CSR/adjacency lists, compressing frontiers, and applying vertex renumbering to improve locality.
Performance notes: For large graphs, direction-optimizing BFS (top-down vs bottom-up) reduces work on high-degree vertices (Beamer et al., 2012). Parallel implementations use distributed frameworks (Pregel/Giraph) or GPU libraries (RAPIDS cuGraph) to process frontiers concurrently. Memory layout and avoiding high-degree hotspots are key engineering concerns.

Applications

Case: Neo4j unweighted shortest path
Code:
// Neo4j 4.x
MATCH (a:Person {id:'A'}), (b:Person {id:'B'})
MATCH p = shortestPath((a)-[*]-(b))
RETURN p

Used in: Neo4j query planner/algorithms (internal BFS-like traversal for shortestPath)
Case: GPU-accelerated BFS with RAPIDS cuGraph
Code:
# cuGraph 22.12+, CUDA 11.x
import cudf, cugraph
df = cudf.DataFrame({'src':[0,1,1],'dst':[1,2,3]})
G = cugraph.Graph()
G.from_cudf_edgelist(df, 'src', 'dst')
res = cugraph.bfs(G, start=0)

Used in: NVIDIA RAPIDS for high-throughput graph analytics
Production patterns:
- Use BFS for single-source unweighted shortest paths / reachability and social graph hops
- Apply direction-optimizing BFS and vertex renumbering on big graphs
- Offload frontier work to GPUs or distribute via Pregel/Giraph for scalability
References: Beamer et al., "Direction-Optimizing Breadth-First Search" (2012); Graph500 implementations.
