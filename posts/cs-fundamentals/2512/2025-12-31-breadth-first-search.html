Breadth-First Search (BFS)

난이도 / Difficulty: Intermediate
날짜 / Date: 2025-12-31

내용

BFS(너비 우선 탐색)는 그래프/트리의 레벨 단위 탐색 알고리즘으로, 최단 경로(가중치 없는 그래프) 계산과 네트워크 탐색에서 실무적으로 중요합니다.

작동 원리: BFS는 큐를 사용해 시작 정점에서 인접 정점을 레벨 순으로 방문합니다. 시간 복잡도는 O(V+E)이며, 방문 표시(visited 비트셋)와 레벨(depth) 배열을 유지해 중복 탐색을 막습니다. 실무에서는 깊이 제한(depth cutoff), 재방문 방지, 그리고 메모리·IO 최적화가 필수입니다.

대규모/분산 환경: 페타바이트급 그래프에서는 단일 머신 메모리에 모든 정점을 올릴 수 없어 외부 메모리 알고리즘(GraphChi)이나 Pregel/Giraph/Pregel 계열의 BSP(초단계 superstep) 모델로 BFS를 병렬화합니다. 또한 두 노드 간 최단경로는 양방향 BFS(bidirectional BFS)로 방문 공간을 기하급수적으로 줄일 수 있습니다.

활용

실제 사용 사례:
- 사례: Neo4j 그래프DB 탐색(Neo4j 4.x)
코드:
// Java API
TraversalDescription td = db.traversalDescription()
  .breadthFirst()
  .relationships(RelationshipType.withName("FRIEND"))
  .maxDepth(3);
for (Path p : td.traverse(startNode)) { /* process */ }

사용처: 소셜 추천, 권한 전파(Neo4j 공식 문서)

- 사례: Apache Giraph / Pregel 스타일 분산 BFS
명령 예: bin/giraph.sh -ca giraph.SplitMasterWorker=false -mc org.apache.giraph.examples.BFSComputation ...
사용처: Facebook/Pregel 논문 기반의 대규모 그래프 처리

실무 패턴:
- 방문 비트셋과 레벨 배열로 메모리 최소화
- 깊이 제한, 양방향 BFS로 탐색 폭 축소
- 분산 환경에선 체크포인트와 superstep 동기화로 회복성 확보

Content

Breadth-First Search (BFS) is a level-order graph traversal used to compute shortest paths in unweighted graphs and to explore network neighborhoods in production systems.

How it works: BFS uses a FIFO queue to expand a frontier one level at a time from a source node. Complexity is O(V+E). Implementations maintain a visited bitset and a distance/level array to avoid revisits and to produce shortest-path distances. Practical systems add depth cutoffs and early exits for performance and safety.

At scale: Single-machine BFS hits memory/IO limits; systems use external-memory algorithms (e.g., GraphChi) or distributed BSP models (Pregel/Giraph/Apache Giraph, Google’s Pregel-inspired systems) where each superstep expands the frontier in parallel. For point-to-point queries, bidirectional BFS is a common optimization to reduce explored nodes dramatically.

Applications

Real-world use case:
- Case: Neo4j graph traversal (Neo4j 4.x)
Code:
TraversalDescription td = db.traversalDescription()
  .breadthFirst()
  .relationships(RelationshipType.withName("FRIEND"))
  .maxDepth(3);
for (Path p : td.traverse(startNode)) { /* handle path */ }

Used for: social recommendations, access propagation (Neo4j docs)

- Case: Apache Giraph / Pregel-style BFS
Command example:
bin/giraph.sh -ca giraph.SplitMasterWorker=false -mc org.apache.giraph.examples.BFSComputation ...
Used for: large-scale graph analytics (Facebook/Pregel lineage)

Production patterns:
- Use visited bitset + level array; prefer bidirectional BFS for single-pair queries
- In distributed runs, checkpoint supersteps and tune frontier partitioning to reduce communication
