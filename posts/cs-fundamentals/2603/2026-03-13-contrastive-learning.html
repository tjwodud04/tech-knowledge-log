Contrastive Learning (InfoNCE) — Retrieval & Embeddings

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-03-13

내용

컨트라스티브 러닝(예: InfoNCE)은 쌍(positive/negative)을 이용해 표현 공간에서 유사도 구조를 학습하여 검색·임베딩 정확도를 높입니다. 프로덕션에서는 빠른 ANN 검색과 결합해 추천·검색·멀티모달 매칭에 쓰입니다.

핵심은 같은 샘플의 두 뷰(augmentation, augmentation된 이미지/문장 또는 쿼리-문서 쌍)를 positive로, 나머지를 negative로 봐 유사도를 최대화(positive)·최소화(negative)하는 손실(예: InfoNCE/NT-Xent)을 최적화하는 것입니다. 수식으로는 p(+|i) = exp(sim(z_i,z_pos)/τ) / Σ_k exp(sim(z_i,z_k)/τ) 같은 softmax 기반 대조 손실이 자주 사용됩니다.
구현적 팁: 대규모 negative가 필요해 MoCo(moco v2)는 모멘텀 인코더 + 큐(queue)로 메모리 뱅크를 구현하고, SimCLR는 매우 큰 배치와 강한 augment를 사용합니다. CLIP(OpenAI)은 이미지-텍스트 쌍에 같은 contrastive objective를 적용해 멀티모달 임베딩을 학습했고, DPR(Facebook AI) 등은 쿼리-문서 비례 검색에 같은 아이디어를 사용합니다. 프로덕션은 보통 in-batch negatives, 하드 네거티브 샘플링, FAISS/Milvus로 ANN 인덱싱, 양자화/IVF로 대규모 서빙을 결합합니다.

활용

사례: SimCLR (Google Research, 2020)
코드: github.com/google-research/simclr (PyTorch) — 학습 예시
python main.py --dataset imagenet --batch-size 1024 --lr 0.3

실무 패턴: 대규모 배치, 온더플라이 augment, 온-디스크 체크포인트(분산 학습)

사례: MoCo (Facebook/Meta)
코드: github.com/facebookresearch/moco — 모멘텀 인코더 + 큐 구현

사례: CLIP (OpenAI) — 이미지-텍스트 매칭으로 검색·필터링에서 활용
코드 예시 (Hugging Face):
from transformers import CLIPProcessor, CLIPModel
model = CLIPModel.from_pretrained('openai/clip-vit-base-patch32')
proc = CLIPProcessor.from_pretrained(...)
emb = model.get_text_features(**proc(texts, return_tensors='pt'))

프로덕션 패턴: FAISS (facebookresearch/faiss v1.7)로 IVF/PQ 인덱싱, 정기적 임베딩 재인덱싱, 배치 및 하드네거티브 기반 재학습(모니터링: recall@k, MRR).

Content

Contrastive learning (e.g., InfoNCE) trains representations by pulling positives together and pushing negatives apart; it's widely used to produce embeddings for retrieval, recommendation, and multimodal matching in production.

Mechanically, you form positive pairs (two augmentations of the same image, query–doc pair, or image–caption) and many negatives. The InfoNCE/NT-Xent loss is common: p(+|i)=exp(sim(z_i,z_pos)/τ)/Σ_k exp(sim(z_i,z_k)/τ), where sim often is cosine. Large negative sets improve discrimination: SimCLR uses very large batch sizes as in-batch negatives; MoCo uses a momentum encoder plus a queue (memory bank) for many negatives; CLIP applies contrastive loss across image-text pairs at web scale.
In deployment, contrastive pretraining is followed by indexing (FAISS/Milvus), approximate search (IVF, PQ, HNSW), and periodic retraining with hard-negative mining. DPR (Dense Passage Retrieval) from Facebook uses similar bi-encoder contrastive training for QA retrieval. Key practical knobs: temperature τ, number/quality of negatives, augmentation or data curation, and ANN index configuration.

Applications

Case: SimCLR (Google Research, 2020)
Source: github.com/google-research/simclr
Command example:
python main.py --dataset imagenet --batch-size 1024 --base-lr 0.3
Production patterns: very large batch/in-batch negatives, distributed mixed-precision training (PyTorch 1.12+/2.0), strong augmentations.

Case: MoCo (Facebook/Meta)
Source: github.com/facebookresearch/moco — uses momentum encoder + queue for memory-efficient negatives

Case: CLIP (OpenAI) used for image-text retrieval
Code (Hugging Face transformers):
from transformers import CLIPProcessor, CLIPModel
m = CLIPModel.from_pretrained('openai/clip-vit-base-patch32')
proc = CLIPProcessor.from_pretrained('openai/clip-vit-base-patch32')
inputs = proc(text=['a cat'], images=[image], return_tensors='pt')
emb = m.get_text_features(**inputs)

Deployment tips: build FAISS index (faiss-cpu 1.7 / faiss-gpu), use IVF+PQ or HNSW for latency/size tradeoffs, monitor recall@k and refresh embeddings when distribution drifts.
