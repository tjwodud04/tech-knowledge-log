Sentence Embeddings & Pooling (e.g., SBERT)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-03-04

내용

문장 임베딩은 문장 단위 의미를 고정 길이 벡터로 표현해 검색, 추천, 클러스터링 등 실무 시스템에서 의미 기반 유사도 계산을 가능하게 한다.

작동 원리: 대표적 접근법인 SBERT(2019)는 두 개의 BERT 인코더를 시암 네트워크로 이용해 문장 쌍 유사도를 빠르게 계산하도록 학습한다. 토큰 레벨 출력(각 토큰의 contextual vector)을 풀링(pooling)해 문장 벡터를 만들며, 풀링 방식은 CLS 토큰, 평균(mean) 풀링, 최대(max) 풀링 등이 있다. 평균 풀링은 attention mask로 비어 있는 토큰을 제외한 가중 평균을 취해 실무에서 가장 안정적이다.

학습·최적화: 실무 모델은 미세조정(fine-tuning)에 contrastive loss, triplet loss, 또는 multiple negatives ranking loss를 사용해 의미적 구조를 보존한다. 생산 환경에서는 GPU 배치 인퍼런스, FP16, 토크나이저 캐시, 그리고 벡터 DB(FAISS, Milvus, Pinecone)와 IVF/PQ 양자화로 지연과 비용을 줄인다.

활용

실제 사용 사례:
- 사례: SBERT 논문 및 SentenceTransformers 라이브러리(UKP Lab) — 논문: Reimers & Gurevych 2019
코드:
# requirements: sentence-transformers==2.2.0, faiss-cpu==1.7.3
  from sentence_transformers import SentenceTransformer, models
  word_emb = models.Transformer('sentence-transformers/all-MiniLM-L6-v2')
  pool = models.Pooling(word_emb.get_word_embedding_dimension(), pooling_mode_mean_tokens=True)
  model = SentenceTransformer(modules=[word_emb, pool])
  sentences = ['search query', 'candidate document']
  embs = model.encode(sentences, convert_to_numpy=True, show_progress_bar=False)
  # FAISS로 인덱싱
  import faiss
  d = embs.shape[1]
  index = faiss.IndexFlatIP(d)  # cosine via normalized vectors
  faiss.normalize_L2(embs)
  index.add(embs)
  D,I = index.search(embs, k=5)

실무 패턴:
- batch encoding + GPU/FP16로 처리(대규모 코퍼스 재인덱싱 시)
- FAISS IVF+PQ 또는 Milvus로 비용/검색속도 균형
- 정기 재학습(로그 기반 hard negatives 수집) 및 벡터 모니터링(거리 분포)
참고: FAISS는 Meta(구 Facebook)에서 개발된 벡터 라이브러리, SentenceTransformers는 널리 쓰이는 구현체.

Content

Sentence embeddings convert sentences into fixed-length vectors for semantic similarity, enabling search, recommendation, clustering, and downstream ML in production systems.

How it works: SBERT (Sentence-BERT, Reimers & Gurevych 2019) uses siamese BERT encoders to produce sentence vectors that allow efficient similarity computation. Token-level outputs from a transformer are reduced to a single vector by pooling—common strategies are CLS token, mean pooling over token vectors, or max pooling. Mean pooling with attention mask (exclude padding) is often most robust in production.

Training & performance: Models are fine-tuned with contrastive losses (e.g., multiple negatives ranking loss, triplet loss) to preserve semantic relationships. Production deployments optimize throughput via batched GPU inference, FP16, tokenizer caching, and use vector stores (FAISS, Milvus, Pinecone) with IVF/PQ quantization to lower latency and storage costs.

Applications

Real-world case:
- Case: SentenceTransformers (UKP Lab) & SBERT paper (Reimers & Gurevych, 2019). FAISS (Meta) used for vector search.
Code:
# pip install sentence-transformers==2.2.0 faiss-cpu==1.7.3
  from sentence_transformers import SentenceTransformer, models
  word = models.Transformer('sentence-transformers/all-MiniLM-L6-v2')
  pool = models.Pooling(word.get_word_embedding_dimension(), pooling_mode_mean_tokens=True)
  model = SentenceTransformer(modules=[word, pool])
  embs = model.encode(['query','doc'], batch_size=64, convert_to_numpy=True)
  # FAISS index
  import faiss, numpy as np
  faiss.normalize_L2(embs)
  idx = faiss.IndexFlatIP(embs.shape[1])
  idx.add(embs)
  D,I = idx.search(embs, k=10)

Production patterns:
- Use IVF+PQ and quantization for billion-scale corpora (FAISS/Milvus)
- Periodic re-embed with new model versions and use mined hard negatives for fine-tuning
- Serve embeddings via batched GPU inference or offload to vector DBs (Pinecone, Milvus) to scale
