Beamforming & Microphone-Array Signal Processing

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-03-18

내용

빔포밍은 여러 마이크 입력을 공간적으로 합성해 특정 방향의 신호를 증폭하고 간섭·잡음을 억제하는 기술로, 음성인식·회의·스마트스피커의 원거리 음성 캡처에서 필수적입니다.

작동 원리: 기본적 빔포밍은 마이크 간 시간지연(도착차)을 보정해 동일 신호를 정렬한 뒤 합쳐 SNR을 높입니다(Delay-and-Sum). MVDR(최소분산왜곡무향)나 LCMV처럼 통계적 방법은 잡음 공분산 추정으로 특정 방향 신호는 유지하면서 잡음을 최소화합니다. 최근 연구·산업에선 신경망 기반 마스크 추정 후 MVDR를 적용하는 'mask-based MVDR'가 실무에서 널리 쓰입니다.
실무 고려사항: 실시간 환경에선 계산량·지연과 하드웨어(ARM DSP, FPGA) 제약이 중요합니다. 따라서 제품에서는 경량화된 시간도메인 비율-합성(beamform weights)이나 on-device filterbank + 서버 기반 NN 보정의 하이브리드 파이프라인을 사용합니다.

활용

실제 사용 사례:
- 사례: Amazon Alexa / Google Nest (far-field voice capture)
코드: Kaldi/CHiME recipes에서 BeamformIt 사용 예
예: beamformit/BeamformIt -f scp:wav.scp outdir/
(CHiME Kaldi egs에서 다중 채널 전처리 파이프라인으로 사용)
- 사례: 연구·오픈소스 (mask-based MVDR)
코드(프로토타입, Python):
# Delay-and-sum prototype (Python, librosa)
  import numpy as np, librosa
  X = [librosa.stft(x, n_fft=512) for x in signals]
  X = np.stack(X, -1)  # freq x frames x mics
  freqs = librosa.fft_frequencies(sr=sr, n_fft=512)[:,None]
  tau = compute_tdoa(doa, mic_positions)  # seconds
  steering = np.exp(-1j*2*np.pi*freqs*tau)
  Y = np.sum(X * steering[None,:,:], axis=2)
  y = librosa.istft(Y)

실무 패턴: WPE(역잔향)→NN 기반 마스크 추정→MVDR 적용(클라우드/서버 연산) 혹은 경량 DS on-device. Kaldi+BeamformIt은 CHiME/AMI 데이터셋 전처리(음성 인식 파이프라인)에 표준으로 쓰입니다.
참고: mask-based MVDR는 Google/학계의 음성분리 연구에서 채택됨(딥 마스크 추정 + 고전적 beamformer 결합).

Content

Beamforming uses multi-microphone arrays to spatially filter sound—boosting signals from a desired direction while suppressing interference—critical for far‑field ASR, conferencing, and smart speakers.

How it works: The simplest is Delay-and-Sum (align channels by TDOA and sum). More advanced schemes use spatial covariance estimates (MVDR, LCMV) to minimize noise power while preserving target response. In modern systems, a neural network predicts time-frequency masks for target/noise and those masks are used to compute covariance matrices for mask-based MVDR (combining learning and classical beamforming).
Practical constraints: latency, compute (DSP vs. CPU/GPU), and array geometry matter. Production systems often pipeline dereverberation (WPE) → mask estimation (NN) → MVDR for best tradeoff, or run cheap DS on-device and heavier mask-MVDR in cloud.

Applications

- Case: Amazon Alexa, Google Nest (far-field capture in production)
