Griffin-Lim Algorithm (Phase Reconstruction for Audio/Vocoders)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-03-16

내용

Griffin-Lim은 단일 채널의 스펙트럼 크기(앰플리튜드)만으로 시간 도메인 파형을 재구성하는 반복적 위상 추정 알고리즘으로, TTS/보코더의 빠른 프로토타이핑과 디버깅에서 널리 쓰입니다.

원리: STFT로 얻은 복소 스펙트럼의 위상 정보가 없을 때, Griffin-Lim은 초기 무작위 또는 제로 위상 추정에서 시작해 역STFT→재-STFT 사이클을 반복하면서 스펙트럼 크기 제약을 만족하도록 위상을 점진 갱신합니다(Griffin & Lim, 1984). 알고리즘은 최소 제곱 목적함수(원신호의 STFT와 재구성 STFT 사이 오차)를 반복적으로 줄입니다.
실무적 포인트: n_fft, hop_length, win_length, 윈도우 타입과 반복 횟수(n_iter)가 품질/지연을 결정합니다. 낮은 반복수는 금속성(phasiness) 아티팩트를 남기고, 높은 반복수(32~100)는 품질을 개선하지만 비용이 늘어납니다. 대부분의 생산 환경은 음질을 위해 신경 보코더(WaveNet, WaveGlow, ParallelWaveGAN 등)로 전환했지만, Griffin-Lim은 경량한 CPU 기반 대체 또는 디버깅 파이프라인으로 여전히 유용합니다.

활용

실제 사용 사례:
- 사례: Tacotron 데모(연구: Wang et al., 2017) 및 여러 오픈소스 TTS 파이프라인에서 멜-스펙트로그램→파형 변환의 기본 방법으로 사용
코드:
import librosa
  # mel: (n_mels, T) 예측 멜 스펙트로그램
  S = librosa.feature.inverse.mel_to_stft(mel, sr=22050, n_fft=1024)
  y = librosa.griffinlim(S, n_iter=60, hop_length=256, win_length=1024)

(librosa >= 0.6; 권장 0.8.1)
- 사례: PyTorch/torchaudio에서 배치화된 변환
코드:
import torch, torchaudio
  gl = torchaudio.transforms.GriffinLim(n_fft=1024, n_iter=60, hop_length=256)
  waveform = gl(magnitude_tensor)  # torchaudio >= 0.7

실무 패턴과 베스트 프랙티스:
- 개발/디버그용 또는 리소스 제약 시 사용; 프로덕션 음질은 신경 보코더로 대체
- STFT/ISTFT 파라미터를 학습/추론 파이프라인과 엄격히 일치시키기
- 품질 향상: n_iter 32~100, 충분한 n_fft(1024~2048), 적절한 윈도우/중첩 사용
참고: Griffin & Lim (1984), Tacotron (2017).

Content

Griffin-Lim is an iterative phase-reconstruction algorithm that converts magnitude-only STFT (e.g., predicted spectrograms) back to time-domain audio; it's lightweight and often used for prototyping or fallback vocoding.

How it works: Starting from an initial complex spectrum (often random phase or zero phase), Griffin-Lim alternates inverse STFT to waveform and forward STFT to update phase while enforcing the target magnitude. Mathematically it performs projection onto the set of signals whose STFT magnitude matches the given spectrogram, iteratively reducing reconstruction error (Griffin & Lim, 1984).
Practical considerations: The main knobs are n_fft, hop_length, window, and n_iter. More iterations (typ. 32–100) improve perceptual quality but increase latency and CPU cost. Modern production TTS systems (Google Tacotron demos, NVIDIA demos) have moved to neural vocoders (WaveNet, WaveGlow, ParallelWaveGAN) for higher fidelity, but Griffin-Lim remains useful for fast CPU-only synthesis, debugging model outputs, or where licensing/size prohibits neural models.

Applications

Real-world use case:
- Case: Tacotron demo and many research/OSS TTS pipelines used Griffin-Lim for demo audio (Wang et al., 2017). Libraries provide battle-tested implementations.
Code (librosa, recommended v0.8.1):
import librosa
  S = librosa.feature.inverse.mel_to_stft(mel_spec, sr=22050, n_fft=1024)
  y = librosa.griffinlim(S, n_iter=60, hop_length=256, win_length=1024)

- Case: torchaudio transform (torchaudio >= 0.7) for batched GPU/CPU pipelines
import torch, torchaudio
  gl = torchaudio.transforms.GriffinLim(n_fft=1024, n_iter=60, hop_length=256)
  waveform = gl(mag_tensor)

Production patterns:
- Use Griffin-Lim as a fallback or debug tool; migrate to neural vocoders (WaveNet by DeepMind, WaveGlow by NVIDIA, ParallelWaveGAN) for production speech services.
- Ensure STFT params match end-to-end training/inference, tune n_iter for latency/quality tradeoff, and pre/post-process (pre-emphasis, denoising) to reduce artifacts.
References: Griffin & Lim (1984); Tacotron (2017); librosa/torchaudio docs.
