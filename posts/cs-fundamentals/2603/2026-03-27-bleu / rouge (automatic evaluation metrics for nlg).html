BLEU & ROUGE — Practical NLG Evaluation

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-03-27

내용

BLEU와 ROUGE는 기계번역·요약 등 NLG(자연어생성) 시스템의 자동 평가 지표로, 빠른 반복 개발과 A/B 비교에 실무적으로 중요합니다.

작동 원리: BLEU는 n-그램 정밀도(precision)를 기반으로 여러 참조(reference) 문장과의 n-그램 겹침 비율을 계산하고, 짧은 출력 페널티(Brevity Penalty)를 적용합니다. 일반적으로 BLEU-4(1~4그램)와 로그 평균을 사용합니다. ROUGE는 요약 평가에 특화되어 ROUGE-N(리콜 기반 n-그램), ROUGE-L(최장 공통 부분열, LCS) 등을 제공합니다. ROUGE는 보통 참조 대비 얼마나 많이 캡처했는지(리콜)에 중점을 둡니다.
실무 포인트: 두 지표는 코퍼스 수준(corpus-level) 점수가 더 안정적이며 문장 수준(sentence-level)은 노이즈가 큽니다. 토크나이제이션, 소문자화, 정규화(숫자/구두점 처리), 평활(smoothing) 방식에 따라 점수가 크게 달라지므로 sacreBLEU처럼 서명(sacred signature)을 남겨 재현성을 확보하세요. 또한 최신 연구·제품에서는 자동지표와 사람평가를 함께 사용합니다(예: 요약 품질, 상호참조 일관성 판단 등).

활용

실제 사용 사례:
- 사례: Fairseq / Facebook AI Research에서 모델 개발 시 BLEU를 사용해 기계번역 모델을 빠르게 비교하고, Hugging Face와 Pegasus 논문에서 ROUGE를 요약 성능 평가에 사용
- 코드:
# sacrebleu 설치 및 corpus-level BLEU (sacrebleu 2.3.1 호환 사용 예)
pip install sacrebleu==2.3.1
sacrebleu reference.txt -i hypothesis.txt -m bleu -b --tokenize=13
# Hugging Face evaluate로 ROUGE
pip install evaluate
python - <<PY
import evaluate
rouge = evaluate.load('rouge')
refs = ['The cat is on the mat.']
hyps = ['Cat sits on mat.']
print(rouge.compute(predictions=hyps, references=refs))
PY

- 실무 패턴:
- 항상 동일한 토크나이저/평활 설정을 기록(sacrebleu signature)
- 코퍼스 단위 평가 우선, 문장단위는 보조 지표로 사용
- 자동 지표 한계(문체·사실성 판단 부족)를 보완하기 위해 소규모 인간평가를 병행
참고: Pegasus(2020)·BART(2019) 요약 논문과 Fairseq의 MT 파이프라인은 각각 ROUGE/BLEU를 표준 벤치마크로 사용합니다.

Content

BLEU and ROUGE are automatic metrics for NLG (MT, summarization) used for fast iteration, model selection, and A/B tests in production and research.

How it works: BLEU computes precision of overlapping n-grams between hypothesis and one-or-more references, applies a brevity penalty to penalize overly short outputs, and aggregates via geometric/log averages (BLEU-4 is common). ROUGE focuses on recall: ROUGE-N counts n-gram overlaps by recall, and ROUGE-L uses longest common subsequence (LCS) to assess fluency-preserving overlaps. Both operate at corpus-level for stability; sentence-level scores are noisy.
Practical details: Tokenization, case-normalization, punctuation handling, and smoothing methods change scores substantially. Use standardized tooling (sacrebleu for BLEU, Hugging Face evaluate or rouge-score for ROUGE) to ensure reproducibility. In production, teams combine automatic metrics with targeted human evaluation because BLEU/ROUGE poorly capture factuality, coherence, and style.

Applications

Real-world use case:
- Case: Used in FAIR/Fairseq for MT (BLEU) and in PEGASUS/BART papers for summarization (ROUGE)
- Code:
# sacrebleu (example, compatible with sacrebleu v2.3.1)
pip install sacrebleu==2.3.1
sacrebleu ref.txt -i hyp.txt -m bleu -b --tokenize=13

# Hugging Face evaluate for ROUGE
pip install evaluate
python - <<PY
import evaluate
rouge = evaluate.load('rouge')
refs = ['The quick brown fox jumps over the lazy dog.']
hyps = ['Quick brown fox jumped over lazy dog.']
print(rouge.compute(predictions=hyps, references=refs))
PY

- Production patterns:
- Record metric config (tokenizer, smoothing, version) — sacreBLEU signatures are widely used in papers and pipelines (Fairseq, OpenNMT examples).
- Use corpus-level BLEU/ROUGE for model selection; validate with human annotation for quality aspects not covered by n-gram overlap (faithfulness, hallucination).
References: BLEU original paper (Papineni et al., 2002), ROUGE (Lin, 2004); sacreBLEU and Hugging Face evaluate are common implementations in industry/research.
