NLG Evaluation: BLEU, ROUGE, BERTScore (practical guide)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-03-30

내용

BLEU/ROUGE/BERTScore는 기계번역·요약·생성모델의 자동 평가 핵심 도구로, 프로덕션 모델 검증·모니터링에서 사람 검토를 보완합니다.

작동 원리: BLEU는 n-그램 정밀도와 길이 페널티(BP)를 결합해 시스템 출력과 참조간 n-그램 중복을 측정합니다(문서 수준 코퍼스-스코어가 안정적). ROUGE는 특히 요약에서 중복된 n-그램(ROUGE-N), 롱레스트 공통 부분(ROUGE-L)을 통해 정보 회수(Recall) 중심 지표를 제공합니다. BERTScore는 사전학습된 Transformer(예: BERT) 임베딩을 사용해 토큰간 의미적 유사도를 매칭하여 의미적 일치를 평가하므로 동의어·재표현에 강합니다.
실무 포인트: 일관된 토크나이저/소문자화, 참조 전처리, corpus-vs-sentence 집계 선택이 결과에 큰 영향을 줍니다. 프로덕션에서는 SacreBLEU(버전 표기 포함)로 재현 가능한 BLEU, google/rouge_score 혹은 pyrouge로 요약 평가, bert-score 패키지로 의미기반 점수를 병행해 사용합니다. 통계적 유의성은 paired bootstrap 또는 chrF와 함께 확인합니다.

활용

실제 사용 사례:
- 사례: Hugging Face + Transformers 실험/CI
코드:
"""
# sacrebleu (reproducible)
pip install sacrebleu==2.3.1
python -c "import sacrebleu; refs=[['The cat.']]; hyps=['A cat.']; print(sacrebleu.corpus_bleu(hyps, refs))"

# rouge (google-rouge)
pip install rouge-score==0.1.2
python -c "from rouge_score import rouge_scorer; s=rouge_scorer.RougeScorer(['rouge1','rougeL'], use_stemmer=True); print(s.score('A cat sleeps','The cat is sleeping'))"

# bert-score
pip install bert-score==0.3.11 transformers==4.30.0
python -c "from bert_score import score; P=['A cat.']; R=['The cat.']; print(score(P,R,lang='en'))"
"""
사용처/버전: Hugging Face 예제 실험/모니터링 파이프라인(Transformers 4.x, Evaluate/SacreBLEU 연동). 연구: BLEU (Papineni et al., 2002), ROUGE (Lin, 2004), BERTScore (Zhang et al., 2020).

실무 패턴:
- CI에 sacrebleu 서명 저장 및 기준화(validation set)
- 여러 메트릭 사용(정밀도/재현율/의미적 유사성) 및 부트스트랩 신뢰구간 보고
- 언어별 토크나이저 고정(특히 중국어/한국어)와 참조 버전 관리

Content

BLEU, ROUGE, and BERTScore are standard automatic metrics for MT, summarization, and NLG; they are used in research and production monitoring to complement human evaluation.

How it works: BLEU computes corpus-level n-gram precision with a brevity penalty to penalize overly short outputs. ROUGE focuses on recall-like overlap measures—ROUGE-N for n-gram overlap and ROUGE-L for longest common subsequence—so it's favored in summarization. BERTScore maps tokens to contextual embeddings from pretrained Transformers (BERT/RoBERTa) and matches token embeddings with cosine similarity, yielding precision/recall/F1 that captures semantic similarity beyond surface n-grams.
Practical details: use reproducible implementations (sacrebleu for BLEU to lock tokenization and signatures), google/rouge_score or pyrouge for ROUGE, and the bert-score package for embedding-based scoring. Decide corpus vs sentence aggregation; report confidence intervals with paired bootstrap; maintain consistent tokenization and reference preprocessing in CI to avoid metric drift. BERTScore is more robust for paraphrase-heavy tasks but is costlier (GPU/embedding), so teams often run it on validation subsets.

Where used / references: BLEU (Papineni et al., 2002) widely used at Google/Microsoft for MT benchmarking; ROUGE (Lin, 2004) used in BART/T5 summarization papers and production pipelines; BERTScore (Zhang et al., 2020) used in academic evaluations and adopted in research-heavy teams for semantic validation.
