Conditional Random Fields (CRFs) for Sequence Labeling

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-03-06

내용

CRF는 조건부 확률 모델로서 연속된 레이블(예: 개체명, 품사)을 문맥 전체를 고려해 예측해 실무 시스템에서 일관된 시퀀스 출력을 보장합니다.

작동 원리: CRF는 입력 시퀀스 x가 주어졌을 때 전체 레이블 시퀀스 y의 조건부 확률 P(y|x)를 모델링합니다. 로컬 특성(단어, 형태소, 접미사, 대문자 등)을 선형 결합한 점수에 대해 경로 전체 점수를 합산하고, 정규화로 확률을 얻습니다. 학습은 L-BFGS, SGD, 또는 확률적 경사 하강법으로 최대우도 추정을 수행하며, 추론은 Viterbi 알고리즘(최고 점수 경로)으로 합니다.
현업 관점: 전통적 NLP 파이프라인(Stanford NER, CRF++)에선 풍부한 수작업 특성(feature engineering)이 핵심입니다. 요즘은 BiLSTM-CRF 구조(Huang et al. 2015, Lample et al. 2016)가 주류로, 신경 임베딩으로 특성을 자동화하고 CRF를 출력층에 둬 레이블 간 제약(예: I-ORG가 B-PER 뒤에 올 수 없음)을 보장합니다. CRF는 작은 데이터에선 여전히 견고하고, 규칙과 특성 결합이 쉬워 실무에서 디버깅·규정 준수 관점에서 유리합니다.

활용

실제 사용 사례:
- 사례: Stanford NER (CRF 기반) — 연구/학계에서 널리 쓰임 (Finkel et al., 2005). 코드: Stanford NER 패키지(예: stanford-ner 3.9.2)로 모델 학습/태깅.
코드 예시 (파이썬 sklearn-crfsuite 사용):
pip install sklearn-crfsuite

from sklearn_crfsuite import CRF
  crf = CRF(algorithm='lbfgs', c1=0.1, c2=0.1, max_iterations=100)
  crf.fit(X_train, y_train)       # X_train: list of token feature dicts
  preds = crf.predict(X_dev)

사용처: 연구·프로토타입, 일부 레거시 파이프라인에서 품사/개체명 인식에 사용.
- 사례: BiLSTM-CRF 패턴 — 논문/프레임워크에서 채택 (Huang et al. 2015). 구현: AllenNLP의 ConditionalRandomField 모듈, Flair의 SequenceTagger(CRF 출력층).
실무 패턴: (1) 임베딩+BILSTM로 특성 추출, (2) CRF로 레이블 제약 적용, (3) L2 규제와 조기종료, (4) Viterbi 디버깅 로그 보관. 참조: Lafferty et al. 2001, Sutton & McCallum 2012.

Content

A CRF models conditional probabilities of label sequences given inputs, producing globally-consistent sequence labels (e.g., NER, POS) useful in production for constrained outputs.

How it works: CRFs score entire label sequences y given input x via feature functions f_k(y_t, y_{t-1}, x, t) and weights w_k; P(y|x) ∝ exp(w·F(x,y)). Training maximizes conditional likelihood (regularized), using L-BFGS or SGD; inference uses Viterbi to find the highest-scoring path. The model enforces label transition constraints and lets overlapping input features influence joint predictions.
Practical flow: In classic ML pipelines engineers hand-craft features (orthography, gazetteers, prefix/suffix). In modern systems the CRF often sits on top of learned representations: e.g., BiLSTM-CRF (Huang et al. 2015, Lample et al. 2016) where BiLSTM provides token scores and CRF enforces valid tag sequences. CRFs remain valuable when data is limited or when explicit structure/constraints are required.

Applications

Real-world use case:
- Case: Stanford NER (CRF-based) — widely used in research (Finkel et al., 2005). Tools: Stanford NER jar (e.g., 3.9.2).
Code:
# install a Python wrapper
  pip install sklearn-crfsuite

from sklearn_crfsuite import CRF
  crf = CRF(algorithm='lbfgs', c1=0.1, c2=0.1, max_iterations=100)
  crf.fit(X_train, y_train)
  preds = crf.predict(X_test)

Where used: prototyping, legacy production NLP pipelines, academic baselines.
- Case: BiLSTM-CRF in production research stacks — used in sequence tagging toolkits (AllenNLP's ConditionalRandomField, Flair's SequenceTagger with CRF). Pattern: use pretrained embeddings or transformers for features, add CRF as output layer to enforce transition constraints; regularize with c1/c2 or dropout and validate with Viterbi-decoded outputs. References: Lafferty et al. 2001; Huang et al. 2015; Sutton & McCallum 2012.
