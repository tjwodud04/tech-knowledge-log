Hidden Markov Models (HMM)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-04-29

내용

숨겨진 마르코프 모델(HMM)은 관찰 가능한 시퀀스 뒤에 숨은 상태 시퀀스를 확률적으로 모델링해 시계열 추론·정렬·분류에 쓰이는 핵심 도구입니다. 생산시스템에서 음성인식, 유전자 서열 검색, 품사 태깅 등에 실무 적용이 많습니다.

작동 원리: HMM은 상태 전이확률 행렬 A, 상태별 관찰확률 B(또는 출력 분포), 초기 상태 분포 pi로 정의됩니다. 주요 알고리즘은 Viterbi(최적 상태 경로 디코딩), Forward-Backward(사후 확률 계산), Baum-Welch(EM 기반 파라미터 추정)입니다. 실무에서는 로그 확률로 연산해 언더플로우를 막고, 정규화·스무딩·정방향-후방향 벡터화로 성능을 끌어올립니다.

산업·학술 적용: 바이오인포매틱스의 profile HMM(예: HMMER)은 단백질 도메인 검색에서 표준이며(Sean Eddy), 음성인식 분야는 전통적으로 HMM+GMM, 이후 HMM+DNN(하이브리드)으로 발전해 Kaldi와 같은 툴킷에서 사용됩니다. 실무에서는 작은 상태 수의 해석 가능 모델이 필요할 때 HMM을 선택하고, 대규모 음성/텍스트는 DNN 또는 end-to-end로 보완합니다.

활용

사례: HMMER (profile HMM) — 생물정보학 도메인 검색
- 출처: HMMER (Eddy), 권장 버전 예: v3.3.2
- 명령:
# 서열 데이터에서 도메인 검색
hmmscan --tblout results.tbl Pfam-A.hmm seqs.fasta

- 사용처: UniProt/도메인 어노테이션 파이프라인에서 대규모 스캐닝

사례: Kaldi 음성 인식 — HMM + DNN 하이브리드
- 출처: Kaldi 오픈소스 (https://github.com/kaldi-asr/kaldi)
- 명령 예시(간단화):
# 단일 작업 예: monophone 모델 학습
steps/train_mono.sh --nj 20 data/train data/lang exp/mono

- 사용처: 연구·기업 프로토타입에서 음성 디코딩 파이프라인 구성

코드: Python(pomegranate v0.14) 간단 HMM
from pomegranate import HiddenMarkovModel, NormalDistribution
model = HiddenMarkovModel()
model.add_states(NormalDistribution(0,1), NormalDistribution(5,1))
model.add_transition(model.start, model.states[0], 0.6)
model.add_transition(model.start, model.states[1], 0.4)
# ... 전이/종료 추가
model.bake()
model.fit([sequence_list])
states = model.predict(sequence)

실무 패턴:
- 로그-스케일 계산, underflow 방지
- 모델 선택: BIC/AIC로 상태 수 결정
- 하이브리드: HMM 구조 + DNN 출력(음성)으로 성능 확장
- 프로덕션: 배치 스캔(hmmscan), 병렬화/스트리밍 디코딩(Kaldi)

Content

Hidden Markov Models (HMMs) model sequences by assuming observations are emitted from hidden states following Markovian transitions; they are widely used for temporal inference, alignment, and decoding in production systems like speech recognition and biosequence analysis.

How it works: An HMM is defined by transition matrix A, emission distributions B (discrete or continuous), and initial distribution pi. Core algorithms are Viterbi (most likely state path), Forward-Backward (posterior probabilities), and Baum–Welch (EM for parameter estimation). Practical implementations operate in log-space, use vectorized forward/backward passes, and incorporate smoothing and regularization to avoid degenerate estimates.

Real-system usage: Profile HMMs (HMMER) are standard for protein domain detection; speech stacks historically used HMM+GMM and later HMM+DNN hybrids (Kaldi) for production ASR. In modern pipelines HMMs still serve where interpretable state sequences or explicit timing constraints are needed; for large-scale neural systems they appear as alignment layers or priors.

Applications

Case: HMMER (profile HMM) for biosequence search
- Source: HMMER (Sean Eddy), common version: v3.3.2
- Command:
hmmscan --tblout results.tbl Pfam-A.hmm seqs.fasta

- Where used: UniProt/Pfam annotation pipelines for large-scale domain scanning

Case: Kaldi ASR — HMM + DNN hybrid
- Source: Kaldi repo (https://github.com/kaldi-asr/kaldi)
- Command example (typical training script):
steps/train_mono.sh --nj 20 data/train data/lang exp/mono

- Where used: research and production prototyping for ASR systems

Code: Python (pomegranate v0.14) quick HMM
from pomegranate import HiddenMarkovModel, NormalDistribution
m = HiddenMarkovModel()
# add gaussian states, transitions, bake
m.fit([seqs])
path = m.predict(seq)

Production patterns:
- Use log-probabilities, numerical stability, and vectorized Baum–Welch
- Model selection with AIC/BIC; regularize emissions
- Hybridize HMM structure with neural nets for emissions (Kaldi-style) or use HMMs for fast batch scanning (HMMER) when interpretability and domain models matter
