Named Entity Recognition (NER)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-04-10

내용

NER은 텍스트에서 사람, 장소, 조직, 날짜 등 실세계 개체를 자동으로 인식·분류하는 기술로, 검색, 지식그래프, 문서 분류 등 프로덕션 시스템에서 핵심 기능입니다.

작동 원리: NER은 시퀀스 라벨링 문제로 BIO/BILOU 같은 태깅 규약을 사용합니다. 전통적으로 HMM/CRF(예: Lafferty의 CRF) 기반 특징 엔지니어링이 쓰였고, 연구·생산은 BiLSTM-CRF(Lample et al., 2016)에서 Transformer(BERT, Devlin et al., 2018)로 이동했습니다. 실제 시스템에서는 서브워드 토크나이저(BPE/WordPiece)로 인한 스팬 정합(spans alignment) 문제를 해결해야 합니다.
운영 관점: 프로덕션에선 모델 압축(distillation), 온디맨드 배치 vs 스트리밍 인퍼런스, 불확실성 임계값(precision/recall trade-off) 조정, 엔티티 연결(entity linking)을 결합합니다. 오픈소스·클라우드(예: spaCy, Hugging Face Transformers, Google Cloud Natural Language, Amazon Comprehend)는 엔터프라이즈에서 바로 활용 가능한 파이프라인을 제공합니다.

활용

사례: Hugging Face + BERT 기반 NER
코드:
# requires: transformers>=4.0
from transformers import pipeline
nlp = pipeline("ner", model="dslim/bert-base-NER", tokenizer="bert-base-cased", grouped_entities=True)
print(nlp("Barack Obama was born in Hawaii."))

사용처: dslim/bert-base-NER(Transformers hub), 연구/프로덕션에서 BERT 파인튜닝 기반 엔티티 추출에 사용
사례: spaCy v3 모델 학습
코드(명령):
# spaCy >=3.0
python -m spacy init config config.cfg --model en_core_web_trf
python -m spacy train config.cfg --output ./output --paths.train ./train.spacy --paths.dev ./dev.spacy

실무 패턴: BIO 태그와 토큰-스팬 정합 처리, 엔티티 정규화+링크(knowledge base) 분리, 온프레미스 배치 추출과 경량 서빙(ONNX/TorchScript) 병행. 연구참조: Lample et al. 2016 (BiLSTM-CRF), Devlin et al. 2018 (BERT), Akbik et al. 2018 (Flair).

Content

NER finds and classifies real-world entities (people, locations, orgs, dates) in text; it's fundamental for search, knowledge graphs, document processing, and downstream ML features in production.

How it works: NER is a sequence-labeling task typically using BIO/BILOU schemes. Traditional systems used feature-engineered CRFs; modern pipelines use neural architectures such as BiLSTM-CRF and Transformers (BERT). Key engineering issues include subword tokenization alignment (WordPiece/BPE), span aggregation, and handling nested/overlapping entities.
Operational details: Production systems optimize for latency and accuracy via model distillation, quantization, batching, and caching. They integrate entity linking (KB resolution), confidence thresholds for downstream consumers, and monitoring for model drift. Popular toolchains in real deployments include spaCy, Hugging Face Transformers, and cloud services (Google Cloud Natural Language, Amazon Comprehend).

Applications

Case: BERT-based NER with Hugging Face
Code:
# requires: transformers>=4.0
from transformers import pipeline
nlp = pipeline("ner", model="dslim/bert-base-NER", tokenizer="bert-base-cased", grouped_entities=True)
print(nlp("Barack Obama was born in Hawaii."))

Used in: dslim/bert-base-NER on the Transformers Hub; common for fine-tuning BERT for NER in production.
Case: spaCy v3 training and serving
Code/commands:
# spaCy >=3.0
python -m spacy init config config.cfg --model en_core_web_trf
python -m spacy train config.cfg --output ./output --paths.train ./train.spacy --paths.dev ./dev.spacy

Production patterns: handle token→span alignment for subword models, separate extraction and KB-linking stages, serve distilled/quantized models (ONNX/TorchScript) for low-latency APIs, and run periodic re-training using labeled logs. Research refs: Lample et al. 2016 (BiLSTM-CRF), Devlin et al. 2018 (BERT), Akbik et al. 2018 (Flair).
