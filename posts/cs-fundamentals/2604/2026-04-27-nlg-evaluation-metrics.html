NLG Evaluation Metrics (BLEU, ROUGE, BERTScore, BLEURT, MoverScore)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-04-27

내용

자연어 생성(NLG) 평가 지표는 모델 출력의 품질(정확성, 유창성, 의미 보존)을 자동으로 수치화해 프로덕션 파이프라인에서 모델 비교·모니터링에 필수적입니다.

핵심 지표들(BLEU/ROUGE)은 n-gram 중복 기반으로 빠르게 대규모 코퍼스 수준에서 성능을 비교합니다. 하지만 의미적 유사성이나 패럴프레이징에는 약하므로 임베딩 기반(BERTScore, MoverScore)이나 학습 기반(BLEURT) 지표가 보완합니다. 산업에서는 배치별(코퍼스) 점수와 샘플별(문장) 분포를 함께 모니터링하고, A/B 테스트·사람 평가와 정기 교차검증하여 자동 지표의 신뢰도를 보정합니다.

기술적으로는: BLEU는 정규화(토크나이즈, lowercasing), n-gram precision과 brevity penalty로 구성됩니다. ROUGE는 요약에서 recall 중심(ROUGE-L은 longest common subsequence). BERTScore는 레이어별 임베딩을 매칭해 F1을 계산하고, BLEURT는 세밀한 오류를 학습한 순위/회귀 모델입니다. 프로덕션에서는 sacrebleu(버전 고정)로 재현 가능한 BLEU, Hugging Face evaluate로 여러 지표 파이프라인화, 그리고 임베딩 캐싱으로 속도 최적화를 적용합니다.

활용

실제 사용 사례:
- 사례: Google/지속적 MT 평가, WMT(공식)에서 BLEU 표준 사용(사전 토크나이제이션 기준)
코드:
# sacrebleu CLI (권장: sacrebleu==2.x 고정)
  echo "system output" > hyp.txt
  echo "reference text" > ref.txt
  sacrebleu ref.txt < hyp.txt

- 사례: Hugging Face Evaluate 라이브러리를 통한 CI 통합(오픈소스/스타트업에서 사용)
코드:
pip install evaluate sacrebleu bert_score rouge-score
  import evaluate
  bleu = evaluate.load('sacrebleu')
  bert = evaluate.load('bertscore')
  preds = ['the cat is on the mat']
  refs = [['the cat is on the mat']]
  print(bleu.compute(predictions=preds, references=refs))
  print(bert.compute(predictions=preds, references=[r[0] for r in refs], lang='en', model_type='roberta-large'))

- 실무 패턴: 지표 버전 고정(sacrebleu), 코퍼스-레벨과 샘플 분포 동시 모니터링, 자동 지표+사람 평가 혼합, 임베딩 모델 캐싱(throughput 최적화), 도메인별 재학습된 BLEURT/thresholding으로 알람 설정
참고: BLEURT는 Google Research(2020) 제안, BERTScore/MoverScore는 의미적 일치 평가에 널리 인용됩니다.

Content

NLG evaluation metrics quantify generated text quality (accuracy, fluency, semantic preservation), enabling model comparison, CI gating, and regression detection in production systems.

Classic metrics (BLEU/ROUGE) rely on n-gram overlap and are fast for corpus-level comparisons but miss semantic paraphrase. Embedding-based metrics (BERTScore, MoverScore) and learned scorers (BLEURT) capture meaning and finer errors. In practice, teams combine automatic metrics with periodic human evaluation and use metric calibration for specific domains.

Technically: BLEU computes n-gram precisions with a brevity penalty after standardized tokenization (sacrebleu enforces reproducibility). ROUGE focuses on recall and LCS for summarization. BERTScore aligns token embeddings (cosine) to get precision/recall/F1; BLEURT is a pre-trained regression model fine-tuned on human judgments. Production considerations include deterministic tokenization, metric/version pinning, embedding caching, and computing both corpus- and sentence-level statistics for drift detection.

Applications

Real-world use cases:
- Case: Machine Translation (WMT / Google / Microsoft historically use BLEU for benchmarks)
Code:
# sacrebleu CLI (pin sacrebleu==2.x for reproducibility)
  echo "system output" > hyp.txt
  echo "reference text" > ref.txt
  sacrebleu ref.txt < hyp.txt

- Case: CI/monitoring with Hugging Face Evaluate (used by many startups and OSS projects)
Code:
pip install evaluate sacrebleu bert_score rouge-score
  import evaluate
  bleu = evaluate.load('sacrebleu')
  bert = evaluate.load('bertscore')
  preds = ['the cat is on the mat']
  refs = [['the cat is on the mat']]
  print(bleu.compute(predictions=preds, references=refs))
  print(bert.compute(predictions=preds, references=[r[0] for r in refs], lang='en', model_type='roberta-large'))

- Production patterns: pin metric implementations (sacrebleu version), normalize/tokenize consistently, combine automatic signals with small-scale human eval, cache embedding computations for BERTScore, retrain or fine-tune BLEURT on domain judgments for alerts. References: BLEURT (Google Research, 2020), BERTScore (Zhang et al.), MoverScore — all commonly used in research and industrial NLG pipelines.
