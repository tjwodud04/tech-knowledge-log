Scaling Laws for Neural Networks

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-04-13

내용

스케일링 법칙은 모델 파라미터 수, 데이터 토큰 수, 계산량이 성능(예: 크로스엔트로피 손실)에 미치는 경험적 관계를 설명하며, 대규모 모델 설계·비용 최적화에 직접적 영향을 줍니다.

작동 원리: 스케일링 법칙은 Kaplan et al. (2020) 등이 보고한 경험적 멱법칙(power law)입니다. 손실 L(N, D) ≈ A N^{-α_N} + B D^{-α_D} + C 형태로 근사하고, 계산량 C(=FLOPs)은 모델 크기 N과 토큰수 D에 따라 결정됩니다. Hoffmann et al. (2022, Chinchilla)는 동일한 계산 예산에서 모델 크기를 줄이고 더 많은 토큰으로 학습하는 것이 효율적이라고 제시해 ‘compute-optimal’ 비율(Tokens-per-param ≈ 20)을 권고했습니다.

실무 관점: 실 시스템에서는 스케일링 법칙을 통해 예산(C)을 주었을 때 최적의 파라미터 수와 데이터량을 추정해 하드웨어·비용 계획을 세웁니다. 대규모 트레이닝은 DeepSpeed ZeRO, FairScale, Hugging Face Accelerate 같은 도구와 NVidia A100/TPU v4 클러스터에서 실행되어 파라미터·메모리 제약을 해결합니다. 또한 로그-라인 피팅으로 α 계수를 추정해 향후 확장 효과를 예측합니다.

활용

실제 사용 사례:
- 사례: OpenAI/DeepMind 연구(예: Kaplan et al. 2020, Hoffmann et al. 2022)에서 스케일링 법칙으로 모델·데이터 설계 가이드 확보
- 사례: Microsoft DeepSpeed(ZeRO)로 대규모 모델을 실제 GPU 클러스터에 배치(프로덕션/연구) — DeepSpeed v0.8+, PyTorch 2.x

코드:
# 간단한 Chinchilla-style 추정(파라미터 N, 토큰 T 최적비례 추정)
compute_budget = 1e25  # FLOPs
# 경험적 규칙: N_opt ~ sqrt(C) , T_opt ~ sqrt(C) => ratio ~ const
import math
N_opt = int(math.sqrt(compute_budget))
T_opt = int(math.sqrt(compute_budget))
print('N_opt', N_opt, 'T_opt', T_opt)


DeepSpeed 실행 예 (CLI):
deepspeed --num_gpus=8 train.py --deepspeed_config ds_config.json

(ds_config.json에 "zero_optimization": {"stage":3})

실무 패턴:
- 먼저 소형/중형 스윕으로 α 계수를 추정하고 예산 기반 N/T 설계를 계산
- ZeRO/FSDP로 메모리 병렬화, 데이터 파이프라인(Streaming dataset)으로 더 많은 토큰 처리
- 비용·성능 트레이드오프를 문서화해 클라우드 예산과 스팟 인스턴스 전략에 반영

Content

Scaling laws describe empirical power-law relations between model size, dataset size (tokens), compute, and performance; they guide cost-efficient design of large models in production and research.

How it works: Empirical scaling laws (Kaplan et al., 2020) fit loss as approximate power laws in parameters N and data D: L(N,D) ≈ a N^{-α_N} + b D^{-α_D} + const. From this, compute-optimal tradeoffs are derived because training FLOPs depend on N and tokens. Hoffmann et al. (2022, Chinchilla) re-evaluated earlier trends and showed that for a fixed compute budget, smaller models trained on more tokens can outperform larger undertrained models — motivating a practical tokens-per-parameter heuristic.

Practical mechanics: Engineers estimate exponents α by small-scale sweeps, then project optimal N and T for a given compute budget. In production, scaling laws inform procurement (GPUs/TPUs), dataset collection vs. model growth, and when to invest in infrastructure like ZeRO (DeepSpeed) or FSDP to enable larger N without OOMs. Tools: PyTorch 2.x, DeepSpeed v0.8+, Hugging Face Transformers v4.x, TPU v4 are common in these pipelines.

Applications

Real-world use case:
- Case: Research sources — Kaplan et al. 2020 (scaling laws), Hoffmann et al. 2022 (Chinchilla). Used by OpenAI/DeepMind to set training targets.
- Case: Microsoft/OSS — DeepSpeed (ZeRO) is used to train large models in production (DeepSpeed v0.8+, PyTorch 2.x).

Code:
# simple heuristic: compute-optimal rough estimate (Chinchilla-style)
compute = 1e25
import math
N_opt = int(math.sqrt(compute))  # rough proportional rule
T_opt = int(math.sqrt(compute))
print('N_opt',N_opt,'T_opt',T_opt)

DeepSpeed launch (example):
deepspeed --num_gpus=8 train.py --deepspeed_config ds_config.json
# ds_config.json: {"zero_optimization": {"stage": 3}}


Production patterns/best practices:
- Run small sweeps to fit α_N and α_D before extrapolating.
- Prefer streaming datasets and checkpointing to increase effective tokens.
- Use ZeRO/FSDP for memory scaling; monitor cost-per-accuracy and retrain vs bigger-model trade-offs.
References: Kaplan et al. 2020, Hoffmann et al. 2022, DeepSpeed docs.
