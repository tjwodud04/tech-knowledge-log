Adversarial Robustness

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-04-20

내용

적대적 견고성은 모델이 의도적으로 설계된 입력(작은 퍼텐셜 변경)에 의해 오작동하지 않도록 하는 기술로, 보안·자율주행·의료 등 실서비스에서 신뢰성과 안전을 확보하는 데 중요합니다.

핵심 원리: 입력 공간에서 작은 노이즈(ℓ_p 제약)를 추가한 공격(input perturbation)이 모델 예측을 크게 바꿀 수 있다는 관찰에서 출발합니다. 대응책은 크게(1) 적대적 학습(Adversarial Training) — 공격을 반복 생성해 모델이 그에 강해지도록 학습, (2) 검출/정규화 — 입력이나 내부 표현의 불일치로 공격을 탐지, (3) 인증된 방어(Verified/Certified Robustness) — 수학적으로 특정 노름 반경 내에서 예측 불변을 보장하는 방법(PGD, randomized smoothing)입니다.
실무 관점: Madry 등(2017)의 PGD 기반 적대적 학습은 실무에서 널리 쓰이며, 물리적 공격(예: 교통 표지 변조)은 자율주행 연구(예: Eykholt et al., 2018)에서 실제 문제로 입증되었습니다. 오픈소스 툴(IBM ART, Foolbox, CleverHans)은 공격·방어 파이프라인을 제공합니다.

활용

사례: Madry et al. (PGD adversarial training)
코드:
# PyTorch: PGD inner-loop (개념적)
def pgd_attack(model,x,y,eps=8/255,alpha=2/255,steps=7):
  x_adv = x + torch.rand_like(x)*1e-6
  for _ in range(steps):
    x_adv.requires_grad_()
    loss = F.cross_entropy(model(x_adv), y)
    grad = torch.autograd.grad(loss, x_adv)[0]
    x_adv = (x_adv + alpha*grad.sign()).clamp(x-eps, x+eps).detach()
  return x_adv
# 학습 루프에서 x_adv로 손실 계산(모델: PyTorch >=1.7, Python 3.8+)

실무 패턴:
- ‘온더플라이’ 공격 생성으로 배치별 적대적 학습(메모리/속도 트레이드오프)
- 검증시에는 AutoAttack/CW/PGD 같은 여러 공격을 사용하여 과적합 방지
참조/툴: IBM Adversarial Robustness Toolbox (ART, github.com/Trusted-AI/adversarial-robustness-toolbox), Foolbox, Madry et al. (2017), Eykholt et al. (2018).

Content

Adversarial robustness is ensuring models resist deliberately crafted small input perturbations that change predictions; it's critical for security-sensitive production systems (perception, fraud detection, medical AI).

Core idea: adversarial examples are small-norm perturbations (ℓ_p-bounded) that cause misclassification. Practical defenses fall into (1) adversarial training — augmenting training with adversarial examples (Madry et al., 2017 PGD), (2) detection & input transformations — e.g., denoising, randomized smoothing, (3) certified defenses — providing provable robustness guarantees for a perturbation radius via methods like randomized smoothing or interval bound propagation. Algorithmically, adversarial training solves a minimax: min_θ E_x [ max_{||δ||≤ε} loss(f_θ(x+δ), y) ], where inner max approximated by PGD/FW attacks.
In production, teams combine adversarial training with ensemble evaluation and robust augmentation; open-source libraries provide ready-to-run attacks/defenses.

Applications

Case: PGD adversarial training (Madry et al.) and toolkits
Code:
# conceptual PyTorch PGD (requires PyTorch>=1.7, Python3.8+)
for x,y in loader:
  x_adv = pgd_attack(model,x,y,eps=8/255,alpha=2/255,steps=7)
  loss = criterion(model(x_adv), y)
  loss.backward(); optimizer.step()

Case: IBM Adversarial Robustness Toolbox (ART)
pip install adversarial-robustness-toolbox
python -c "from art.attacks.evasion import FastGradientMethod"

Production patterns:
- Use multiple attacks (AutoAttack, PGD, CW) for validation
- Balance robustness vs. clean accuracy (curriculum or mixed batches)
- Deploy certified smoothing for high-assurance components (e.g., perception pipelines in autonomous-driving research such as Eykholt et al. 2018)
References: Madry et al. 2017, IBM ART (github.com/Trusted-AI/adversarial-robustness-toolbox), Foolbox/CleverHans.
