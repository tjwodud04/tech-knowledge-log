Syntactic Dependency Parsing

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-06-01

내용

의존 구문 분석(dependency parsing)은 문장 내 단어들 간의 문법적 관계(주어-동사, 목적어-동사 등)를 트리/그래프 형태로 찾는 기술로, 정보추출·질의응답·기계번역의 전처리와 특징으로 실무에서 널리 사용됩니다.

작동 원리: 실무에서 쓰이는 알고리즘은 크게 전이(transition)-기반(예: arc-standard, arc-eager)과 그래프(graph)-기반(예: MST)으로 나뉩니다. 전이 기반은 스택과 버퍼 조작으로 O(n) 평균 속도를 내고 실시간 파이프라인에 적합한 반면, 그래프 기반(특히 non-projective 지원)은 모든 가능한 arc를 점수화해 Chu-Liu/Edmonds 같은 알고리즘으로 MST를 찾아 정확도를 높입니다. 현대의 SOTA는 BiLSTM으로 토큰 컨텍스트를 인코딩한 뒤 biaffine score(Dozat & Manning, 2017)로 head–dependent 점수를 구해 O(n^2) 연산으로 트리를 구성합니다.
운영 고려사항: 언어별 UD(Universal Dependencies) 트리뱅크로 사전학습/파인튜닝하고, 비문법/도메인 특화 텍스트는 라벨 스케마 재학습 또는 pseudo-projective 변환을 적용합니다. 실무에서는 경량 모델(spaCy)과 고정확 모델(Stanza/Stanford CoreNLP, Biaffine 구현)을 용도에 따라 혼합 배포합니다.

Content

Dependency parsing finds grammatical relations between words (head–dependent) as a tree/graph. It's crucial in production pipelines for IE, QA, MT, and NLP feature extraction.

How it works: Practically, parsers split into transition-based (arc-standard, arc-eager) and graph-based (MST) methods. Transition models operate with a stack and buffer and can achieve near O(n) runtime—suitable for streaming pipelines. Graph-based parsers score all possible arcs and use algorithms like Chu-Liu/Edmonds to recover non-projective trees. Modern state-of-the-art uses BiLSTM encoders to produce contextualized token vectors and a biaffine scorer (Dozat & Manning, 2017) to compute head-dependent scores; decoding is typically O(n^2).
Operational notes: Use Universal Dependencies (UD) treebanks for pretraining and fine-tune for domain-specific text. Non-projective constructions may require MST decoding or pseudo-projective transforms. Production setups commonly mix lightweight parsers (spaCy) for low-latency needs and higher-accuracy models (Stanford Stanza, Biaffine implementations) for batch jobs.

Applications

Real-world use case:
- Case: spaCy (Explosion AI) used in many production NLP systems for fast dependency parsing
Code:
pip install spacy==2.3.7
  python -m spacy download en_core_web_sm

import spacy
  nlp = spacy.load('en_core_web_sm')
  doc = nlp('Google released BERT in 2018.')
  for tok in doc:
      print(tok.text, tok.dep_, tok.head.text)

- Case: Stanford CoreNLP / Stanza (Stanford NLP Group) used in research and backend pipelines
Code:
# CoreNLP v4.x server
  java -Xmx4g -cp "*" edu.stanford.nlp.pipeline.StanfordCoreNLPServer -port 9000

- Case: Deep Biaffine Parser (Dozat & Manning, 2017) — reference SOTA in many papers; implementations in AllenNLP/PyTorch
Production patterns:
- Use lightweight parser for real-time, heavyweight Biaffine for batch/analytics
- Fine-tune on UD or in-domain annotated data; use MST or pseudo-projective fixes for non-projectivity
- Monitor parse quality on domain data; fall back to heuristics if parse confidence low
