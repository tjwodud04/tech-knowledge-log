Fine-tuning & Efficient Adaptation (LoRA, Adapter Tuning)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-06-22

내용

LoRA와 어댑터 튜닝은 전체 모델 가중치를 업데이트하지 않고 저비용으로 모델을 도메인/태스크에 맞추는 기법으로, 대형 언어모델을 프로덕션에 효율적으로 배포할 때 핵심 수단입니다.

작동 원리: LoRA는 Transformer의 특정 가중치 행렬에 대해 저랭크 보정행렬(A·B)을 추가하고 오직 이 보정행렬만 학습합니다. 어댑터 튜닝은 각 레이어 내부에 작은 MLP 블록(어댑터)을 삽입해 이 블록만 업데이트합니다. 두 방법 모두 수백만→수천~수십만 파라미터만 학습해 저장·전송·앙상블이 쉬워집니다.

기술적 세부: LoRA는 학습 시 원래 가중치 W를 고정하고 W + BA를 사용합니다(랭크 r, 스케일 α). 어댑터는 down-proj (d→m)→nonlinearity→up-proj (m→d) 구조를 사용하며, 여러 태스크별 어댑터를 모델에 병렬·직렬로 추가해 멀티태스크를 처리합니다. 실무에서는 PEFT(Hugging Face), AdapterHub/adapter-transformers 라이브러리, bitsandbytes(8-bit)와 accelerate로 메모리/속도 최적화를 결합합니다.

활용

실제 사용 사례 + 코드
출처: LoRA (Hu et al., 2021), AdapterHub, Hugging Face PEFT (github.com/huggingface/peft).

LoRA 예 (Hugging Face PEFT):
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, get_peft_model

model = AutoModelForCausalLM.from_pretrained('meta-llama/Llama-2-7b')
config = LoraConfig(r=8, lora_alpha=32, target_modules=['q_proj','v_proj'])
model = get_peft_model(model, config)
# train only LoRA params with accelerate/bitsandbytes

Adapter 예 (Adapter-Transformers):
from transformers import AutoModelWithHeads, AutoTokenizer
from transformers.adapters import AdapterConfig
model = AutoModelWithHeads.from_pretrained('bert-base-uncased')
conf = AdapterConfig.load('pfeiffer', reduction_factor=16)
model.add_adapter('taskA', config=conf)
model.train_adapter('taskA')

실무 패턴: 기본 모델은 freeze, PEFT 체크포인트(작은 파일)만 배포, 여러 어댑터/LoRA를 런타임에 로드해 멀티테넌시·A/B 테스트. r, reduction_factor, lr, merge옵션을 실험해 성능·메모리 균형 조정.

Content

LoRA and adapter tuning let you adapt large models by training small added parameters instead of full weights—critical for cost-efficient production fine-tuning, multi-tenant models, and fast iteration.

How it works: LoRA injects low-rank update matrices into selected weight projections (W + BA) and only trains A and B (rank r). Adapter tuning inserts small bottleneck MLPs inside layers (down-proj→nonlinearity→up-proj) and learns those blocks. Both reduce trainable parameters from billions to thousands–millions, enabling frequent, cheap fine-tuning and storage of many task-specific adapters.

Technical notes: Typical LoRA configs use r=4..16 and a scaling α; target_modules are often attention q/k/v projections. Adapters use reduction_factor (d→d/reduction_factor) and can be stacked or routed per task. In production, combine PEFT APIs (Hugging Face), AdapterHub, bitsandbytes 8-bit weights, and accelerate for memory and speed gains.

Applications

Real-world examples + code
Sources: LoRA (Hu et al., 2021), AdapterHub (Pfeiffer et al.), Hugging Face PEFT (github.com/huggingface/peft), adapter-transformers (github.com/adapterhub/adapter-transformers).

LoRA (Hugging Face PEFT):
from transformers import AutoModelForCausalLM
from peft import LoraConfig, get_peft_model

model = AutoModelForCausalLM.from_pretrained('meta-llama/Llama-2-7b')
config = LoraConfig(r=8, lora_alpha=32, target_modules=['q_proj','v_proj'])
model = get_peft_model(model, config)
# use accelerate + bitsandbytes to train only LoRA params

Adapter (Adapter-Transformers):
from transformers import AutoModelWithHeads
from transformers.adapters import AdapterConfig
model = AutoModelWithHeads.from_pretrained('bert-base-uncased')
conf = AdapterConfig.load('pfeiffer', reduction_factor=16)
model.add_adapter('taskA', config=conf)
model.train_adapter('taskA')

Production patterns: freeze base weights; keep small checkpoints per adapter/LoRA; load adapters dynamically for multi-tenant inference; tune r/reduction_factor and learning rates; monitor calibration when combining with retrieval or RLHF.
