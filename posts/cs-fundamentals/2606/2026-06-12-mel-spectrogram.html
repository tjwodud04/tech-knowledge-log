Mel-Spectrogram

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-06-12

내용

멜-스펙트로그램은 사람의 청각 특성을 반영한 주파수 축 변환으로, 음성/오디오 모델에서 입력 피처로 널리 사용됩니다. 실무에서는 모델 일관성, 계산 효율, 그리고 재현 가능한 전처리 파이프라인이 중요합니다.

멜-스펙트로그램은 원시 오디오를 STFT로 시간-주파수 변환한 뒤 멜 필터뱅크(사람 청각의 비선형 주파수 해상도)를 적용해 주파수 축을 압축한 행렬입니다. 보통 파워 스펙트럼(p=2)을 사용한 뒤 로그(또는 dB) 스케일을 적용해 동적 범위를 줄입니다. 핵심 파라미터는 sample_rate, n_fft, hop_length, win_length, n_mels, fmin/fmax이며, 이 값을 학습/추론 환경에서 동일하게 유지해야 모델이 일관된 입력을 받습니다.

실무적 세부사항으로는 pre-emphasis(고주파 강조), 정규화(utterance-wise mean-variance 또는 dataset-level), silence trimming, 그리고 음성 합성/인식에서의 역변환(그리핀-림즈, 멜-투-파워 네트워크) 처리가 있습니다. GPU 배치 처리를 위해 torchaudio/NVIDIA 구현을 사용하면 속도와 메모리 효율이 좋습니다.

활용

실제 사용 사례 — Google Tacotron2 (샘플: Tacotron2+WaveGlow)과 NVIDIA의 Tacotron2 구현에서 mel 스펙트로그램을 합성 모델의 입력/조건 벡터로 사용합니다(논문: “Natural TTS from Tacotron 2”). 오픈소스 예: librosa와 torchaudio 사용 코드:

Python (librosa):
import librosa, numpy as np
y, sr = librosa.load('audio.wav', sr=22050)
S = librosa.feature.melspectrogram(y=y, sr=sr, n_fft=1024, hop_length=256, n_mels=80, fmin=80, fmax=8000)
logS = librosa.power_to_db(S, ref=np.max)

PyTorch (torchaudio, GPU):
import torchaudio
melspec = torchaudio.transforms.MelSpectrogram(sample_rate=22050, n_fft=1024, hop_length=256, n_mels=80, f_min=80, f_max=8000).to(device)
logS = torchaudio.transforms.AmplitudeToDB()(melspec(waveform))

실무 베스트프랙티스: 파라미터 고정, 로그 스케일 사용, 배치 단위 GPU 연산, 저장은 float32/numpy로, 학습시 데이터 증강(시간마스킹, 주파수마스킹) 적용. 참조: NVIDIA tacotron2 repo, Google Tacotron2 paper, librosa docs.

Content

A mel-spectrogram is a time–frequency representation mapped to the mel scale to mimic human hearing; it’s a standard input for speech synthesis, ASR, and many audio ML systems. In production you need stable preprocessing and reproducible parameters.

Technically, a mel-spectrogram is computed by taking the STFT of an audio waveform, converting magnitudes (power) through a mel filterbank (triangular filters spaced on the mel scale), and typically applying a log or dB transform to compress dynamic range. Key parameters are sample_rate, n_fft, hop_length, win_length, n_mels, and fmin/fmax — these must be identical for training and inference to avoid domain shift.

Operational details include pre-emphasis filtering, silence trimming, per-utterance or dataset normalization, and optionally converting back to waveform (Griffin-Lim or neural vocoders like WaveGlow/WaveNet). For throughput, use GPU-accelerated libraries (torchaudio, NVIDIA implementations) and compute mel features in batches. Data augmentation patterns (SpecAugment: time/frequency masking) are common in production ASR pipelines.

Applications

Real-world examples — Tacotron2 (Google/NVIDIA) uses mel-spectrograms as conditional inputs for neural vocoders (paper: "Natural TTS from Tacotron 2"). Open-source tools: librosa, torchaudio, TensorFlow’s tf.signal. Example code:

# librosa
import librosa, numpy as np
y, sr = librosa.load('audio.wav', sr=22050)
S = librosa.feature.melspectrogram(y=y, sr=sr, n_fft=1024, hop_length=256, n_mels=80, fmin=80, fmax=8000)
logS = librosa.power_to_db(S, ref=np.max)

# torchaudio (GPU)
import torchaudio
m = torchaudio.transforms.MelSpectrogram(sample_rate=22050, n_fft=1024, hop_length=256, n_mels=80, f_min=80, f_max=8000).to(device)
db = torchaudio.transforms.AmplitudeToDB()
mel = db(m(waveform))

Production patterns: freeze preprocessing parameters, cache computed mels, use float32 storage, apply SpecAugment for robustness. References: Tacotron2 (Google), NVIDIA tacotron2 repo, librosa/torchaudio docs.
