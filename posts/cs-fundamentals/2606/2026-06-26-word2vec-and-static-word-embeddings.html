Word2Vec & Static Word Embeddings

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-06-26

내용

Word2Vec와 정적 임베딩은 단어를 고정 차원 실수 벡터로 매핑해 검색·추천·피처 엔지니어링에서 빠르게 유사성 연산을 가능하게 합니다.

작동 원리: Word2Vec의 대표 모델은 Skip-gram(주변 단어 예측)과 CBOW(중심 단어 예측)이며, 네거티브 샘플링 또는 계층적 소프트맥스를 사용해 확률적 학습을 효율화합니다. 수학적으로 Skip-gram with Negative Sampling(SGNS)은 희소 공기행렬의 암묵적 행렬분해와 동치라는 연구(Levy&Goldberg)가 있어 단순한 통계 기반 기법임에도 강력합니다.
실무 관점: 정적 임베딩은 컨텍스트 무관하지만 계산 비용이 낮아 대량 로그에서 빠르게 벡터를 추출해 피처 스토어에 저장하거나, 약식 의미 유사성 기반 필터링(예: 검색 전처리, 후보군 생성), A/B 실험용 고정 특징으로 널리 사용됩니다. Google News pre-trained, Stanford GloVe, Facebook FastText 같은 공개 모델이 산업계에서 초기 벡터화 표준으로 쓰였습니다.

활용

실제 코드 및 사용처:
1) Gensim으로 학습/로딩(Prod용 오프라인 파이프라인):
from gensim.models import Word2Vec, KeyedVectors
sentences = LineSentence('corpus.txt')
model = Word2Vec(sentences, vector_size=300, window=5, sg=1, negative=5, epochs=5)
model.wv.save_word2vec_format('w2v.bin', binary=True)
# 로드
kv = KeyedVectors.load_word2vec_format('GoogleNews-vectors-negative300.bin', binary=True)

2) FastText(아웃오브보우드 처리, Facebook):
# fastText CLI: 학습
./fasttext skipgram -input corpus.txt -output model -dim 300

3) 서빙/모델 통합 패턴 (PyTorch 예):
import torch.nn as nn
weights = kv.vectors # numpy array
emb = nn.Embedding.from_pretrained(torch.FloatTensor(weights), freeze=False)
# prod: 임베딩을 피처 스토어에 캐시하고 ANN(FAISS)로 유사도 검색

참고: 대용량 시스템에서는 FAISS로 ANN 인덱싱, feature-store(Feast 등)에 벡터 저장, 정기 재학습으로 드리프트 대응. 주요 참고문헌: Mikolov et al. (2013), Pennington et al. GloVe (2014), Bojanowski et al. FastText (2017).

Content

Word2Vec and static embeddings map words to fixed real-valued vectors enabling fast similarity ops for search, recommendation, and feature engineering in production.

How it works: The main Word2Vec flavors are Skip-gram (predict context from a target word) and CBOW (predict target from context). Training uses negative sampling or hierarchical softmax to scale to large corpora. Levy & Goldberg showed SGNS implicitly factorizes a shifted PMI matrix, linking it to matrix-factorization views (hence strong empirical performance despite simplicity).
Practical tradeoffs: static embeddings are context-agnostic but cheap to compute and store, so teams use them for offline feature pipelines, candidate generation, lightweight semantic search, and as initialization for downstream models. Popular pretrained sets include Google News vectors, GloVe (Stanford), and FastText (Facebook) which handles OOV via subword units.

Applications

Real-world code and where it's used:
1) Train/load with gensim (batch offline pipeline):
from gensim.models import Word2Vec, KeyedVectors
model = Word2Vec(LineSentence('corpus.txt'), vector_size=300, window=5, sg=1, negative=5, epochs=5)
model.wv.save_word2vec_format('w2v.bin', binary=True)
kv = KeyedVectors.load_word2vec_format('GoogleNews-vectors-negative300.bin', binary=True)

2) FastText CLI (Facebook production use for OOV robustness):
./fasttext skipgram -input corpus.txt -output model -dim 300

3) Integrate into model serving (PyTorch):
import torch, torch.nn as nn
weights = torch.FloatTensor(kv.vectors)
embedding = nn.Embedding.from_pretrained(weights, freeze=False)
# Use ANN (FAISS) for nearest-neighbor candidate retrieval in production

Where companies use it: Google/academic origin (Mikolov et al.), Facebook uses FastText for upstream text features, and many teams combine static embeddings + FAISS for fast candidate retrieval in ranking/recommendation. Best practices: persist vectors in a feature store, use ANN for scalable NN search, retrain periodically to reduce drift, and prefer FastText or subword models when OOV is common.
