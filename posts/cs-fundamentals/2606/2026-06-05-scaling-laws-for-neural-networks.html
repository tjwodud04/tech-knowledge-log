Scaling Laws for Neural Networks

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-06-05

내용

스케일링 법칙은 모델 크기(파라미터 수), 데이터(토큰 수), 계산량(플롭스)이 성능(손실/정확도)에 미치는 법칙을 경험적으로 기술하며, 대규모 모델 설계·예산 배분에 실무적 가이드라인을 제공합니다.

작동 원리: Kaplan et al. (2020)에서 언어모델의 손실은 모델 크기 N과 데이터량 D에 대해 대체로 멱법칙(power law)을 따릅니다(L ≈ a N^{-α} + b D^{-β}). Hoffmann et al. (Chinchilla, 2022)은 'compute-optimal' 경계를 도출해 주어진 계산 예산 C에서 N과 D의 최적 분배를 제시했습니다(실무적으로 토큰 수 ≈ 20 × 파라미터 수 권장으로 요약됨). 기술적으로는 작은 스케일 실험으로 지수를 회귀(로그-로그 선형회귀)하고 이를 바탕으로 대형 모델의 marginal return을 예측합니다.
실무 적용: 엔지니어는 예산(C)을 정하고, 경험적 지수로부터 손실 감소율을 예측해 모델 크기·데이터 크기를 선택합니다. 또한 학습 스케줄(학습률, 토큰 이터레이션), 체크포인트 빈도, 데이터 파이프라인(샤딩/토큰화)과 결합해 비용-효율을 최적화합니다.

활용

실제 사용 사례:
- 사례: DeepMind (Chinchilla, 2022) — 70B 파라미터 모델을 약 1.4T 토큰으로 재설계해 compute-optimal 전략을 입증
코드: 스케일링 지수 추정(예제)
import numpy as np
# losses measured at different N, D
N = np.array([1e6, 1e7, 1e8])
L = np.array([2.0, 1.5, 1.2])
coef = np.polyfit(np.log(N), np.log(L), 1)
alpha = -coef[0]
print('alpha≈', alpha)

사용처: 모델/데이터 계획 수립 단계에서 실험 결과를 로그-로그 회귀로 외삽해 대규모 설계를 결정(DeepMind, OpenAI 연구실 전략).
- 사례: Hugging Face + DeepSpeed 실무: 스케일링 법칙으로 파라미터 대비 토큰 목표를 정하고 deepspeed config로 배치/accumulation 조정
설정 예:
{"train_micro_batch_size_per_gpu":4, "gradient_accumulation_steps":8}

프로덕션 패턴: 작은 스케일 실험 → 지수 추정 → compute-optimal (N,D) 산정 → 분산 학습(ZeRO/MP)으로 메모리/시간 효율화. 참고: Kaplan et al. 2020, Hoffmann et al. 2022 (Chinchilla).

Content

Scaling laws empirically relate model size (parameters), dataset size (tokens), and compute to performance (loss/accuracy) and provide practical rules for budgeting and architecture decisions in production ML systems.

How it works: Kaplan et al. (2020) showed language-model loss scales roughly as power laws with model size N and data D (L ≈ a N^{-α} + b D^{-β}). Hoffmann et al. (2022, "Chinchilla") derived the compute‑optimal frontier: for a fixed compute budget C you can trade N vs D to minimize loss. Practically, you run small-scale experiments, fit exponents with log–log linear regression, then extrapolate to pick N and D that maximize return on compute.
Technical notes: training compute for transformers scales approximately with N × D × per-token cost; fitting is done via linear regression on (log N, log L) and (log D, log L) to get α, β. Use these to predict marginal gains and to choose dataset size (Hoffmann suggests ≈20 tokens per parameter for compute-optimal regimes). Combine results with infrastructure constraints (GPU memory, IO, ZeRO/MP) when deploying.

Applications

Real-world use cases:
- Case: DeepMind (Chinchilla, 2022) — redesigned model/data (70B params, ~1.4T tokens) to demonstrate compute-optimal training
Code: quick fit for α from small runs
import numpy as np
N = np.array([1e6,1e7,1e8])
L = np.array([2.0,1.5,1.2])
alpha = -np.polyfit(np.log(N), np.log(L), 1)[0]
print('alpha≈', alpha)

Where used: research-driven capacity planning at DeepMind/OpenAI to decide model vs data budgeting.
- Case: Production training stacks (Hugging Face, DeepSpeed, Megatron-LM)
Code/Config: tune tokens-per-param and DeepSpeed run settings
{"train_micro_batch_size_per_gpu":4, "gradient_accumulation_steps":8}

Production patterns: run scaled-down ablations → fit scaling exponents → choose compute-optimal (N,D) → enable memory sharding (ZeRO), mixed-precision, data streaming. Key refs: Kaplan et al. 2020; Hoffmann et al. 2022 (Chinchilla).
