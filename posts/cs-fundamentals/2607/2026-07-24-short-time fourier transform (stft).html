Short-Time Fourier Transform (STFT)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-07-24

내용

STFT는 신호를 짧은 겹침 창(window)으로 나누어 각 창에 FFT를 적용해 시간-주파수 표현을 만드는 기법으로, 실시간 오디오 처리·음성인식·음원분리 등 실무 시스템의 표준 전처리입니다.

작동 원리는 간단합니다: 입력 신호를 윈도우(예: Hamming)로 자르고 n_fft 길이의 FFT를 계산하며, hop_length(오버랩율)를 통해 시간 해상도와 지연(latency)을 조절합니다. 결과로 복소수 스펙트로그램(크기와 위상)이 나오며, 크기는 로그 스펙트럼이나 Mel 필터뱅크로 변환해 ML 입력으로 자주 사용합니다. 역변환은 ISTFT(Overlap-Add)로 위상과 창을 맞춰 재구성하며, 불연속성이나 창 보정(window normalization)에 주의해야 합니다.

실무 관점에서는 n_fft, hop_length, window 선택이 지연-해상도 트레이드오프를 결정합니다. 배치 처리엔 FFTW/cuFFT, GPU에선 cuFFT나 torch.stft를 쓰고, 실시간 시스템(WebRTC, Kaldi의 프론트엔드, Deezer Spleeter 등)은 프레임 버퍼와 링 버퍼로 온라인 STFT를 구현합니다. 위상 기반 처리(복원, 위상 보상)는 음질에 큰 영향을 줍니다.

활용

실제 사용 사례:
- Kaldi ASR: Kaldi는 STFT/Mel 필터를 프론트엔드로 사용(https://kaldi-asr.org).
- WebRTC 노이즈 억제: 온라인 STFT 기반으로 실시간 레이턴시 최적화(https://webrtc.org).
- Spleeter(Deezer): 음악 분리에서 STFT 기반 스펙트로그램을 U-Net에 입력(https://github.com/deezer/spleeter).

파이썬 예제(librosa):
import librosa, numpy as np
y, sr = librosa.load('audio.wav', sr=None)
S = librosa.stft(y, n_fft=1024, hop_length=256, window='hann')
mag_db = librosa.amplitude_to_db(np.abs(S))
# 역변환
y_rec = librosa.istft(S, hop_length=256, window='hann')

PyTorch(배치·GPU):
import torch
x = torch.from_numpy(y).float().to('cuda')
S = torch.stft(x, n_fft=1024, hop_length=256, window=torch.hann_window(1024).to('cuda'), return_complex=True)

실무 팁: n_fft는 2의 제곱 선택, 윈도우 미리 계산, 배치 FFT 사용, 로그 스케일+정규화 적용, 온라인 모드에선 링버퍼와 hop_size로 레이턴시 설정, 위상 보정 필요 시 복소수 네트워크/리컨스트럭션을 고려하세요.

Content

STFT splits a signal into short, overlapping windows and applies FFT per frame to produce a time–frequency representation; it's a cornerstone for audio analysis, speech systems, and real-time DSP.

How it works: STFT applies a window (e.g. Hann) to successive, overlapping frames and computes an FFT for each frame. Key parameters—n_fft (frequency resolution), hop_length (time resolution/latency), and window type—govern the time-frequency trade-off. Output is a complex spectrogram (magnitude + phase); magnitudes are commonly converted to log scale or Mel bands for ML, while phase is needed for high-quality reconstruction via ISTFT (overlap-add with window normalization).

Implementation notes: production systems precompute windows, use FFTW/cuFFT or GPU batched FFTs, and choose power-of-two n_fft for speed. Online/low-latency systems (WebRTC, real-time mixers) maintain a frame ring buffer and compute STFT incrementally. Research and production pipelines often fuse STFT magnitude with neural networks (e.g., U-Nets, conv nets) for separation/enhancement; phase-aware models or complex-valued networks improve reconstruction.

Applications

Real-world references: Kaldi uses STFT/Mel frontend for ASR (https://kaldi-asr.org), WebRTC uses online STFT for noise suppression (https://webrtc.org), Deezer Spleeter feeds STFT spectrograms into a U-Net for source separation (https://github.com/deezer/spleeter).

Python (librosa):
import librosa, numpy as np
y, sr = librosa.load('audio.wav', sr=None)
S = librosa.stft(y, n_fft=1024, hop_length=256, window='hann')
mag_db = librosa.amplitude_to_db(np.abs(S))
y_rec = librosa.istft(S, hop_length=256, window='hann')

PyTorch (GPU, batched):
import torch
x = torch.from_numpy(y).float().unsqueeze(0).to('cuda')
win = torch.hann_window(1024).to('cuda')
S = torch.stft(x, n_fft=1024, hop_length=256, window=win, return_complex=True)

Production best practices: use batched FFT libraries (FFTW/cuFFT), precompute windows, choose n_fft/hop to meet latency targets, normalize log-mel inputs for NN training, and consider phase-aware or complex-domain models when reconstruction quality is critical.
