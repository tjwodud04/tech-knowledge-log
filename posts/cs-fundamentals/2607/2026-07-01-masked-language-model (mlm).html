Masked Language Model (MLM)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-07-01

내용

마스킹 언어모델(MLM)은 입력 토큰 일부를 가리고 해당 토큰을 예측하도록 사전학습하는 방식으로, 컨텍스트를 양방향으로 학습해 검색·랭킹·문서이해 같은 실무 시스템에서 중요한 표현을 만든다.

작동 원리: MLM은 문장 내 일부 토큰을 [MASK]로 바꿔 모델이 원래 토큰을 예측하게 하는 손실(크로스엔트로피)을 최소화한다(예: BERT, Devlin et al. 2018). 실제 구현에서는 동적 마스킹(dynamic masking)으로 학습 데이터마다 마스크를 새로 생성해 과적합을 줄이고, whole-word 또는 span masking(SpanBERT)으로 단어단위/구간 정보를 보존한다. RoBERTa(Liu et al.)는 더 큰 배치·더 긴 학습으로 MLM 품질을 높였고, ELECTRA(Clark et al.)는 'replaced token detection'로 효율을 개선했다.
운영관점: MLM 기반 인코더는 검색 쿼리 이해, 문서 임베딩, 엔티티 추출 등에서 fine-tune 되어 사용된다. 생산 환경에서는 mixed precision, gradient accumulation, 데이터 병렬화로 대규모 프리트레이닝을 수행하고, 추론용으로는 사전학습된 encoder를 경량화(주입, distillation)한다.

Content

Masked Language Modeling (MLM) trains models to predict masked tokens given bidirectional context; it's the backbone for encoder-style models used in search, QA, NER and document understanding in production.

How it works: MLM randomly masks tokens in input and optimizes the model to predict the original tokens (cross-entropy). Classic example: BERT (Devlin et al., 2018). Practical improvements include dynamic masking (new masks each epoch), whole-word or span masking (SpanBERT) to handle multi-token words, and large-batch regimes used by RoBERTa to improve convergence. ELECTRA replaces masked-token prediction with a more sample-efficient replaced-token detection objective.
Operational notes: In real systems pretrained MLM encoders are fine-tuned for downstream tasks (ranking, intent detection, extractive QA). Production training uses distributed data parallelism, gradient accumulation, mixed precision, and often further pretraining on domain-specific corpora. For inference, models are distilled or quantized to meet latency constraints.

Applications

Real-world code and sources: BERT (Devlin et al. 2018), RoBERTa (Liu et al. 2019), SpanBERT, Hugging Face Transformers.
1) Quick mask prediction demo:
from transformers import pipeline
fill = pipeline('fill-mask', model='roberta-base')
fill('The capital of France is .')
2) Pretraining pattern (Hugging Face):
from transformers import AutoTokenizer, AutoModelForMaskedLM, DataCollatorForLanguageModeling, Trainer, TrainingArguments
tokenizer = AutoTokenizer.from_pretrained('bert-base-uncased')
model = AutoModelForMaskedLM.from_pretrained('bert-base-uncased')
collator = DataCollatorForLanguageModeling(tokenizer, mlm=True, mlm_probability=0.15)
# tokenized_dataset should be list of input_ids
trainer = Trainer(model=model, args=TrainingArguments(output_dir='./out', per_device_train_batch_size=16), data_collator=collator, train_dataset=tokenized_dataset)
trainer.train()
Production patterns/best practices: use dynamic/whole-word/span masking depending on downstream tasks, mix MLM with replaced-token detection for efficiency, apply large-batch warmup schedules, enable AMP and model parallelism, and perform distillation/quantization for low-latency serving.
