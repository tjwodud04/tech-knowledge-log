Room Impulse Response & Dereverberation (RIR / Reverberation)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-05-04

내용

룸 임펄스 응답(RIR)은 실내에서 음원이 마이크에 도달할 때 직접음과 반사음의 시간·주파수 특성을 캡처한 필터이며, 현실 세계의 음성 시스템(ASR, 회의 녹음, 보이스 어시스턴트)에서 성능을 좌우합니다.

작동 원리: 실내 음향은 선형 시불변 시스템으로 근사되어 음원 신호 s(t)는 RIR h(t)와의 컨볼루션 x(t)=s(t)*h(t)+n(t)로 모델링됩니다. RIR은 직접음 도달 시간, 초기반사(조기반사) 및 잔향(후반반사)의 에너지 분포를 포함합니다. RIR을 얻는 방법은 실제 측정(일회성 펄스, 엉덩이 사인, ESS)과 시뮬레이션(이미지 소스법, ray-tracing)이 있습니다 — Allen & Berkley의 이미지 소스법은 실무에서 빠른 합성에 널리 쓰입니다.

역처리/디리버베레이션: 전통적 방법은 WPE(Weighted Prediction Error)를 이용한 선형 예측 기반 잔향 제거와 멀티마이크 빔포밍+WPE 파이프라인입니다(Nakatani 등). 최근에는 DNN 기반 스펙트럼 매핑, Conv-TasNet 스타일의 시간영역 네트워크, 또는 GAN 계열의 리버브-리스토어 모델이 널리 연구됩니다. 실무에서는 데이터 증강(예: Kaldi의 reverberate_data_dir.py)으로 모델을 RIR에 강건하게 만들거나, 실시간 애플리케이션에서 저지연 WPE/beamformer를 병행합니다.

활용

실제 사용 사례:
- 사례: Kaldi (ASR 데이터 증강)
코드:
# Kaldi egs 스크립트로 방 시뮬레이션된 데이터 생성
steps/data/reverberate_data_dir.py --rir-set-parameters "min=0.2,max=0.5,rooms=sim" data/train data/train_reverb

사용처: Kaldi egs (ASR 트레이닝 데이터 확장)

- 사례: pyroomacoustics (RIR 합성) 및 nara_wpe (디리버브)
코드:
import pyroomacoustics as pra
room = pra.ShoeBox([6,5,3])
room.add_source([2,1.5,1.0], signal=signal)
room.add_microphone_array(pra.MicrophoneArray([[4],[3.5],[1.2]], room.fs))
room.compute_rir(); rir=room.rir[0][0]

from nara_wpe.wpe import wpe
enhanced = wpe(multi_channel_signal, taps=10, delay=3)

사용처: 연구/프로토타이핑 (pyroomacoustics GitHub, nara_wpe pip)

실무 패턴: 데이터 증강+멀티마이크 처리로 모델 강건화, 측정 RIR을 혼합해 도메인 갭 최소화, 실시간에서는 저지연 WPE나 스택드 빔포머를 사용. 참고: REVERB/CHiME 챌린지 논문과 Kaldi 레시피들이 산업적 레퍼런스.

Content

Room Impulse Response (RIR) is the filter capturing direct and reflected sound from a source to a microphone; modeling and removing reverberation is crucial for far-field speech systems (ASR, conferencing, voice assistants).

How it works: Acoustics in a room are commonly approximated as a linear time-invariant system: observed x(t)=s(t)*h(t)+n(t), where h(t) is the RIR containing direct path, early reflections and late reverberation. RIRs are obtained by measurement (impulse, ESS sine sweep) or simulation (image-source method, ray tracing). The Allen & Berkley image-source method is often used in research and simulation pipelines because it efficiently synthesizes early reflections.

Dereverberation algorithms: Traditional approaches include WPE (Weighted Prediction Error), which models late reverberation as a predictable component and removes it via multichannel linear prediction (Nakatani et al.). Pipelines in production often combine beamforming (MVDR) with WPE. Modern ML approaches use supervised spectral mapping, time-domain networks (Conv-TasNet variants), or GANs trained on simulated + measured RIR-augmented corpora. In practice teams use simulated RIRs for data augmentation (Kaldi), pyroomacoustics for RIR generation, and nara_wpe for dereverb in Python-based workflows.

Applications

Real-world use case:
- Case: Kaldi (ASR data augmentation)
Code:
# Kaldi-style reverberation (example call)
steps/data/reverberate_data_dir.py --rir-set-parameters "min=0.2,max=0.5,rooms=sim" data/train data/train_reverb

Where used: Kaldi recipes for robust ASR training; REVERB/CHiME datasets use similar augmentation.

- Case: pyroomacoustics (image-source RIR) + nara_wpe (dereverb)
Code:
import pyroomacoustics as pra
room=pra.ShoeBox([6,5,3])
room.add_source([2,1.5,1.0], signal=signal)
room.add_microphone_array(pra.MicrophoneArray([[4.,3.5,1.2]]).T, room.fs)
room.compute_rir(); rir=room.rir
# apply rir to clean signal, then
from nara_wpe.wpe import wpe
enh = wpe(multi_channel_signal, taps=10, delay=3)

Where used: pyroomacoustics (GitHub) for prototyping; nara_wpe (pip) implements WPE used in research.

Production patterns: augment training with both measured and simulated RIRs, use multi-mic beamforming + WPE for far-field ASR (Google/Alexa/Azure public talks and CHiME/REVERB literature), and prefer low-latency variants for real-time devices. References: REVERB challenge, CHiME series, Nakatani WPE papers, Kaldi recipes.
