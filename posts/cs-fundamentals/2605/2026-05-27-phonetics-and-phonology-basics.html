Phonetics & Phonology Basics for Speech Systems

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-05-27

내용

음성학(physiology/acoustics)과 음운론(음운 단위와 규칙)은 ASR/TTS/발화정렬 같은 실무 시스템에서 음향-문자 연결을 안정화하는 핵심입니다.

음성학은 음성 신호의 물리적·생리적 특성(성대진동, 포먼트, F0 등)을 다루고, 음운론은 의미 단위를 구분하는 음소(phoneme)와 음운 규칙(동화, 탈락 등)을 다룹니다. 실제 음성 시스템에서는 음성학적 특징(예: MFCC, filterbanks, F0, formants)과 음운적 표상이 결합되어 음성 인식의 입력·라벨을 구성합니다. 전통적 파이프라인은 G2P(grapheme-to-phoneme)로 사전을 만들고 HMM-GMM/TDNN(예: Kaldi)이나 end-to-end 모델(wav2vec 2.0 + CTC/seq2seq)로 매핑합니다.

현장 기술 포인트: 음향 전처리(MFCC/CMVN, voice activity detection), 강제정렬(forced alignment)으로 정밀 라벨 생성, IPA 기반/언어별 G2P로 다국어 확장. 저자원에서는 조음 특성(feature-based phonology)과 self-supervised 학습(wav2vec 2.0, Facebook AI 2020)을 조합하면 성능 향상에 도움이 됩니다.

활용

실제 사용 사례:
- 사례: Kaldi(ASR 연구·프로덕션)에서 MFCC/CMVN 파이프라인
코드:
# Kaldi feature extraction (typical)
compute-mfcc-feats --config=conf/mfcc.conf scp:wav.scp ark:- | compute-cmvn-stats ark:- ark,scp:cmvn.ark,cmvn.scp

- 사례: Montreal Forced Aligner(발화 정렬)로 대규모 라벨 생성
코드:
# MFA align (example)
mfa align /data/corpus /models/english.dict /models/english_acoustic_model /output

- 사례: wav2vec 2.0 (Facebook AI) self-supervised pretraining -> fine-tune for ASR
코드 (PyTorch/librosa):
import librosa
y, sr = librosa.load('utt.wav', sr=16000)
mfcc = librosa.feature.mfcc(y, sr, n_mfcc=13)

실무 패턴: CMVN + VAD, lexicon-based G2P(Phonetisaurus/Epitran), forced-alignment로 고품질 트레이닝 라벨 확보, SpecAugment/augmentation으로 일반화. 참고: Kaldi repo, wav2vec 2.0 (FAIR), Montreal Forced Aligner docs.

Content

Phonetics (acoustics/articulation) and phonology (phoneme inventories/rules) are foundational for robust ASR, TTS, forced alignment and multilingual pipelines.

Phonetics studies physical speech properties (voicing, formants, F0), while phonology models abstract units (phonemes, allophones, feature bundles). Practical systems extract acoustic features (MFCCs, filterbanks, pitch, formants) and map them to phonemic or graphemic targets. Classic pipelines use G2P to build lexica and HMM-GMM/TDNN (e.g., Kaldi) acoustic models; modern systems use self-supervised encoders (wav2vec 2.0) with CTC/seq2seq fine-tuning.

Key engineering points: feature normalization (CMVN), VAD, forced alignment to create timestamped phoneme labels, and IPA/G2P for language coverage. For low-resource languages, combining articulatory features and SSL pretraining often yields better generalization than naive grapheme-only models.

Applications

Real-world use case:
- Case: Kaldi (production research) uses MFCC + CMVN feature pipelines
Code:
# Kaldi feature extraction
compute-mfcc-feats --config=conf/mfcc.conf scp:wav.scp ark:- | compute-cmvn-stats ark:- ark,scp:cmvn.ark,cmvn.scp

- Case: Montreal Forced Aligner (MFA) for large-scale forced alignment
Code:
# MFA align (example)
mfa align /data/corpus /models/english.dict /models/english_acoustic_model /output

- Case: wav2vec 2.0 (Facebook AI, 2020) pretrain -> fine-tune for ASR
Code (Python/librosa):
import librosa
y, sr = librosa.load('utt.wav', sr=16000)
mfcc = librosa.feature.mfcc(y, sr, n_mfcc=13)

Production patterns: maintain lexicon + G2P (Phonetisaurus/Epitran) for robustness, use forced-alignment to label training sets, apply CMVN/VAD and SpecAugment, prefer SSL pretraining (wav2vec 2.0, HuBERT) for low-resource and multilingual deployments. Sources: Kaldi repo, wav2vec 2.0 (FAIR paper 2020), Montreal Forced Aligner.
