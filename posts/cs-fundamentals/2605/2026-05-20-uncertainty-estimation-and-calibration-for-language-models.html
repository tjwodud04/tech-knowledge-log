Uncertainty Estimation & Calibration for Language Models

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-05-20

내용

언어모델의 불확실성 추정과 캘리브레이션은 예측 신뢰도를 정량화해 실서비스(리스크 제어, 라우팅, 인간 검수)에 필수적입니다.

핵심: 캘리브레이션(calibration)은 모델 확률(예: softmax)을 실제정확도와 일치시키는 작업이고, 불확실성 추정은 예측의 신뢰도를 수치적으로 표현합니다. 실무에서 과신(overconfidence) 문제는 자동화된 응답, 의학·법률 등 규제 영역에서 큰 위험을 초래합니다.

기술적 요약: 대표적 방법은 온도 스케일링(temperature scaling, Guo et al. 2017) — validation set에서 온도 T를 최적화해 logits/T에 softmax를 취함. 앙상블(Deep Ensembles, Lakshminarayanan et al. 2017)은 서로 다른 초기화로 여러 모델를 학습해 예측 평균과 분산으로 불확실성 추정. MC-Dropout(Gal & Ghahramani 2016)은 추론시 드롭아웃을 켜고 여러 샘플을 평균해 베이지안 근사. 다른 방법으로는 플랫 스케일링(Platt), 아이소토닉 회귀, 그리고 규범적 보증을 제공하는 conformal prediction이 있습니다.

활용

실제 사용 사례:
- 사례: Hugging Face/Transformers 파이프라인, scikit-learn, 연구 코드
- 코드 (온도 스케일링, PyTorch, Transformers 4.30, PyTorch 2.1):
# simple temperature scaling eval
import torch, torch.nn.functional as F
logits = model(input_ids).logits
T = torch.tensor(1.2, requires_grad=True)
probs = F.softmax(logits / T, dim=-1)
# optimize T on val set by minimizing NLL

- 코드 (scikit-learn CalibratedClassifierCV, scikit-learn>=1.2):
from sklearn.calibration import CalibratedClassifierCV
clf = CalibratedClassifierCV(base_estimator, method='isotonic', cv=5)
clf.fit(X_val, y_val)

- 사례: Deep Ensembles (research repo, Lakshminarayanan et al. 2017) — production pattern은 모델 수(3–10)와 예측 평균·분산 산출. 비용-정합성 고려: 실시간엔 온도 스케일링 + conformal wrapper, 배치엔 앙상블/MC-Dropout 사용.
- 베스트프랙티스: 검증셋으로 캘리브레이션, 모니터링(시간에 따른 캘리브레이션 drift), 불확실도 임계값으로 라우팅(human-in-loop).

Content

Uncertainty estimation and calibration quantify a language model’s confidence; they are crucial in production for risk control, routing, and human-in-the-loop workflows.

Core: Calibration aligns model probabilities (softmax) with empirical accuracy; uncertainty estimation produces numeric confidence (e.g., predictive variance). Overconfident LMs can cause harmful automation in customer support, medical/financial domains, and search/result ranking.

How it works: Temperature scaling (Guo et al., 2017) finds scalar T on validation data minimizing NLL: softmax(logits/T). Deep Ensembles (Lakshminarayanan et al., 2017) train multiple models with different seeds and average logits; predictive variance is an uncertainty signal. MC-Dropout (Gal & Ghahramani, 2016) keeps dropout on at inference and aggregates samples as a Bayesian approximation. Alternatives include Platt/isotonic calibration, conformal prediction for guaranteed coverage, and Bayesian NNs or probabilistic libs (Pyro, TFP) for richer posteriors.

Applications

Real-world use cases:
- Case: Transformers inference + temperature scaling (Hugging Face Transformers 4.30, PyTorch 2.1)
- Code (temperature scaling skeleton):
# apply temperature to logits
logits = model(**batch).logits  # (B, V)
T = torch.tensor(1.1)
probs = torch.softmax(logits / T, dim=-1)

Optimize T on held-out data via NLL.
- Case: scikit-learn calibration API (scikit-learn>=1.2)
from sklearn.calibration import CalibratedClassifierCV
clf = CalibratedClassifierCV(base_estimator, method='isotonic', cv=5)
clf.fit(X_train, y_train)

- Case: Deep Ensembles (research & production pattern) — run N model checkpoints (N=3–10), average logits and compute variance for uncertainty; used broadly in ML systems where compute allows it. Conformal prediction is applied in regulated domains to give formal error bounds.
- Production patterns: cheap calibration (temperature) for low-latency inference, ensembles/MC-Dropout for batch or high-risk paths, monitor calibration drift, and route low-confidence outputs to human review.
