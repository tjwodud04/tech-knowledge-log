Contrastive Learning for Text Representations / 텍스트 표현을 위한 대비학습

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-05-11

내용

대비학습(contrastive learning)은 텍스트 문장들을 임베딩 공간에서 유사도 기준으로 정렬해 검색·분류·군집 성능을 높이는 기법으로, 생산 시스템의 효율적인 Dense Retrieval과 임베딩 기반 서비스에서 핵심 역할을 합니다.

작동 원리: 텍스트 대비학습은 '긍정 쌍'과 '부정 샘플'을 만들어 임베딩을 가깝게/멀게 학습합니다. 대표 손실은 InfoNCE(소프트맥스 기반)로, 온도 파라미터 τ와 정규화된 내적(sim(u,v)/τ)을 사용해 확률을 만듭니다. 주요 전략은 (1) 데이터 증강: dropout 기반 SimCSE(무감독)처럼 동일 문장을 서로 다른 드롭아웃 경로로 긍정 쌍을 생성, (2) 레이블 기반 supervised contrastive(같은 라벨들을 긍정), (3) in-batch negatives나 메모리 큐(MoCo 스타일)로 대규모 음성 샘플 제공, (4) 하드 네거티브 채굴(ANCE, DPR)으로 실제 검색 품질 향상.
실무 팁: 임베딩 L2 정규화, temperature(0.05~0.2) 튜닝, 큰 batch 또는 large negative pool, query/doc 별 인코더 분리(검색) 등이 핵심입니다.

활용

실제 사용 사례:
- 사례: SimCSE (Gao et al., 2021) 연구 및 코드; DPR (Karpukhin et al., 2020) — 페이스북 FAISS 기반 검색 파이프라인에서 in-batch 하드네거티브 활용; ANCE (Microsoft) — Bing에서 대규모 네거티브를 통해 문서 검색 정밀도 개선.
코드:
# sentence-transformers 기반 인-배치 대비학습 예시
# requires: sentence-transformers==2.2.2 torch==1.13.1 transformers==4.31.0
from sentence_transformers import SentenceTransformer, InputExample, losses, models
model = SentenceTransformer('sentence-transformers/distilbert-base-nli-stsb-mean-tokens')
data = [InputExample(texts=[s,s_alt]) for s,s_alt in pairs]
train_loss = losses.MultipleNegativesRankingLoss(model)
model.fit(train_objectives=[(data, train_loss)], epochs=1, batch_size=64)

실무 패턴:
- Retrieval: query/doc 별 인코더로 서빙, FAISS로 ANN 검색(DPR/ANCE 패턴)
- 학습: 큰 배치(또는 메모리 큐), 온도 튜닝, 하드네거티브 재샘플링
출처: SimCSE repo (GitHub), sentence-transformers docs, DPR/ANCE 논문·코드.

Content

Contrastive learning for text trains embeddings so semantically similar texts are close and dissimilar ones are far — critical for dense retrieval, semantic search, and robust sentence embeddings in production.

How it works: Contrastive methods create positive and negative pairs and optimize an InfoNCE-like loss: p(pos|anchor) = exp(sim(a,p)/τ) / sum(exp(sim(a,all)/τ)). Common engineering choices: (1) augmentation — SimCSE (unsupervised) uses dropout-as-augmentation to produce positive pairs; (2) supervised contrastive uses labels to form positives; (3) in-batch negatives or a memory queue (MoCo-style) provide many negatives without storing them all; (4) hard negative mining (DPR/ANCE) improves retrieval precision. Practically, normalize embeddings, tune temperature τ (≈0.05–0.2), use large batch sizes or an explicit negative pool, and separate query/doc encoders for low-latency search.
Implementation notes: use MultipleNegativesRankingLoss (sentence-transformers) or custom InfoNCE loss; for retrieval combine with FAISS/ANN and periodically refresh negative pools.

Applications

Real-world case:
- Case: SimCSE (Gao et al., 2021) — unsupervised/supervised contrastive text embeddings; DPR (Karpukhin et al., 2020) — Facebook Dense Passage Retrieval; ANCE (Microsoft) — approximate nearest negative mining used in web-scale search.
Code:
# train contrastive embeddings with sentence-transformers
# pip: sentence-transformers==2.2.2 transformers==4.31.0 torch==1.13.1
from sentence_transformers import SentenceTransformer, InputExample, losses
model = SentenceTransformer('sentence-transformers/distilbert-base-nli-stsb-mean-tokens')
train_examples = [InputExample(texts=[a,b]) for a,b in pairs]
loss = losses.MultipleNegativesRankingLoss(model)
model.fit([(train_examples, loss)], epochs=1, batch_size=128)

Production patterns:
- Use FAISS for ANN indexing; refresh embeddings when indexing updates (DPR/ANCE pipelines).
- Combine supervised positives and mined hard negatives; monitor embedding drift and re-index.
Sources: SimCSE GitHub/paper, sentence-transformers docs, DPR/ANCE papers and open-source implementations.
