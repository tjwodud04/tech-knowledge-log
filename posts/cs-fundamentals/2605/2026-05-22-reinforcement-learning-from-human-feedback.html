Reinforcement Learning from Human Feedback (RLHF)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-05-22

내용

RLHF는 모델 출력에 대해 사람의 비교·등급 피드백을 보상 신호로 바꿔 언어모델을 튜닝하는 기법으로, 실서비스에서 모델 정렬(alignment)과 안전성 개선에 핵심 역할을 합니다.

작동 원리는 크게 두 단계입니다. 첫째, 인간 라벨러(또는 합성 정책)가 모델 출력 쌍에 대해 선호(비교) 레이블을 수집하고 이를 학습해 보상모델(reward model)을 만듭니다. 둘째, 생성 모델(policy)을 보상모델이 주는 보상을 최대화하도록 강화학습(PPO 등)으로 미세조정합니다. 실무에서는 KL-penalty로 참조 언어모델(reference model)과의 드리프트를 제어하고, 보상모델 과적합 방지를 위해 별도 검증셋과 보상 클리핑을 씁니다.

산업 적용에서 주의할 점은 라벨 품질·분포 편향과 보상 설계의 민감성입니다. 대규모 시스템(예: ChatGPT/InstructGPT)에서는 수만~수십만 건의 비교 라벨과 분산 학습(DeepSpeed, Accelerate)을 사용하며, 보상모델 업데이트 주기, 라벨러 지침, 안전성 히어스틱(콘솔리데이션/필터링)을 운영 파이프라인에 넣습니다.

활용

사례: OpenAI InstructGPT/ChatGPT 논문(2022) — PPO 기반 RLHF로 응답 선호를 최적화
코드: Hugging Face trl 패키지 예시 (실무에서 많이 사용)
from transformers import AutoModelForCausalLM, AutoTokenizer
from trl import PPOTrainer, PPOConfig

tokenizer = AutoTokenizer.from_pretrained('gpt2')
model = AutoModelForCausalLM.from_pretrained('gpt2')
config = PPOConfig(batch_size=4, ppo_epochs=4)
trainer = PPOTrainer(model, tokenizer, config=config)
queries = tokenizer(['Q1','Q2'], return_tensors='pt', padding=True)
responses = model.generate(queries['input_ids'])
rewards = reward_fn(queries, responses)  # reward model outputs
trainer.step(queries['input_ids'], responses, rewards)

출처/프로젝트: GitHub 'huggingface/trl'·'carperai/trlx', OpenAI InstructGPT paper. 실무 패턴: KL 제약(reference LM), reward model 검증셋 분리, human-in-the-loop 라벨링 파이프(라벨 가이드라인·QC), 분산 학습(DeepSpeed/Accelerate) 및 mixed-precision으로 비용 절감. 보상모델 과적합 시 정책이 비생산적 문답을 생성하므로 정기적 재라벨링과 안전 평가가 필수.

Content

RLHF turns human preferences into a learned reward signal and uses RL (commonly PPO) to fine-tune a generative model—key for aligning LLMs to real-world user intent and safety needs.

How it works: collect pairwise or scalar human feedback on model outputs, train a reward model to predict preferences, then optimize the policy (the LM) with a policy-gradient RL algorithm (PPO is standard). Practical systems add a KL penalty to a reference model to prevent distributional drift, use reward clipping/normalization, and validate the reward model on held-out comparisons to avoid overfitting.

In production, pipelines include annotation UIs, QC for labeler consistency, automated pre-filters for unsafe outputs, and iterative retraining of the reward model. Scaling requires distributed training (DeepSpeed/Accelerate), mixed precision, and careful hyperparameter choices: KL coefficient, reward baseline, PPO clip epsilon, entropy bonus, and batch sizes—these all affect stability and cost.

Applications

Case: OpenAI InstructGPT / ChatGPT (OpenAI 2022) — RLHF with PPO + KL penalty
Code: example using Hugging Face trl (used in many open-source experiments)
from transformers import AutoModelForCausalLM, AutoTokenizer
from trl import PPOTrainer, PPOConfig

tokenizer = AutoTokenizer.from_pretrained('gpt2')
model = AutoModelForCausalLM.from_pretrained('gpt2')
cfg = PPOConfig(batch_size=8, ppo_epochs=4)
trainer = PPOTrainer(model, tokenizer, config=cfg)
inputs = tokenizer(['Prompt A','Prompt B'], return_tensors='pt', padding=True)
outs = model.generate(inputs['input_ids'])
rewards = reward_model.score(inputs, outs)
trainer.step(inputs['input_ids'], outs, rewards)

Sources/tools: OpenAI InstructGPT paper (2022); Christiano et al. 'Deep RL from Human Preferences' (2017); Hugging Face 'trl' and CarperAI 'trlx' repos. Production patterns: KL-penalty to reference LM, human-in-the-loop labeling + QC, reward-model holdout and calibration, distributed training (DeepSpeed/Accelerate), and continuous monitoring for reward hacking and safety regressions.
