Disjoint Set Union (Union-Find)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-01-07

내용

Disjoint Set Union(Union-Find)은 서로소 집합의 합집합과 탐색을 빠르게 처리하는 자료구조로, 그래프 연결성, 이미지 분할, SMT(자동정리) 등 실제 시스템에서 대규모 동적 동치관계를 관리하는 데 핵심적입니다.

작동 원리: 각 원소는 트리의 루트(대표자)를 가리키며, find는 경로압축(path compression)으로 트리 높이를 낮추고, union은 랭크(rank)나 크기(size)를 기준으로 작은 트리를 큰 트리에 붙입니다. 이렇게 하면 연산의 상수에 가까운 역아커만(α(n))의 상한을 얻습니다. 구현상 고려사항은 병렬/분산 환경에서의 동시성, 록-프리 또는 배치 병합(batch union) 전략이며, 외부 기억장치 처리(디스크 기반 대규모 데이터)에서는 스트리밍과 외부 정렬과 결합해 사용합니다.
실무 영향: Kruskal 기반의 MST(최소 신장 트리), 이미지 처리의 연결성 레이블링, SMT 솔버의 합동성 폐쇄(congruence closure) 등에서 성능·메모리 병목을 완화합니다.

활용

실제 사용 사례:
- 사례: OpenCV(이미지 연결성 레이블링)
코드: cv::connectedComponentsWithStats(src, labels, stats, centroids); // OpenCV 4.x
실무 패턴: 2-패스 알고리즘에서 로컬 합집합을 먼저 병합 후 전역 path-compression 적용
- 사례: Boost C++(그래프 알고리즘의 Kruskal)
코드: #include
// boost::disjoint_sets ds(&rank[0], &parent[0]);
사용처: Boost.Graph의 Kruskal 구현(Boost 1.70+)
- 사례: Z3/SMT 솔버(동치 관계 관리)
설명: Microsoft Z3의 내부 합동성 폐쇄는 union-find 기반 구조를 사용해 동치 클래스 전파와 재작성 규칙을 효율화(참고: Z3 소스, Microsoft Research).
실무 팁: path-compression + union-by-rank을 기본으로 하고, 멀티스레드 환경에서는 배치 합집합 후 단일 스레드로 압축하거나 락-그레인 합병을 사용하세요.

Content

Disjoint Set Union (Union-Find) is a data structure for maintaining partitioned sets with fast union and find operations; it’s widely used to manage dynamic equivalence classes in production systems like graph libraries, image processors, and SMT solvers.

How it works: each element points to a parent forming trees whose roots are set representatives. Find uses path compression to flatten trees; union uses rank/size to attach smaller trees to larger ones. Combined, these give nearly-constant amortized time per operation (inverse Ackermann α(n)). Practical concerns include concurrency (lock-free or batched merges), external-memory variants for huge datasets, and integration into streaming or parallel graph frameworks.
Where it matters: Kruskal MST implementations, two-pass connected-component labeling in computer vision, and congruence closure/equality propagation in SMT solvers — all rely on union-find to keep costs low while merging many equivalence relations.

Applications

Real-world use case:
- Case: OpenCV connected components (image processing)
Code: cv::connectedComponentsWithStats(src, labels, stats, centroids); // OpenCV 4.x
Production pattern: local unions per tile, then border merges + final path compression to scale on large images.
- Case: Boost (Kruskal in Boost.Graph)
Code: #include
boost::disjoint_sets ds(&rank[0], &parent[0]);
Where used: Boost.Graph MST routines (Boost 1.70+).
- Case: Z3 SMT solver (Microsoft Research)
Note: Z3’s congruence-closure/equality engine uses union-find-like structures to propagate equalities efficiently (see Z3 source on GitHub).
Production patterns: always use path-compression + union-by-rank, batch unions in parallel settings, and prefer per-shard local DSU then global merge for distributed workloads.
