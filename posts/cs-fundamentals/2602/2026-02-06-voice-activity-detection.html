Voice Activity Detection (VAD)

난이도 / Difficulty: Intermediate
날짜 / Date: 2026-02-06

내용

VAD(음성 활동 검출)는 오디오 스트림에서 말이 있는 구간을 판별해 대역폭, ASR 비용, 음성 인식 정확도를 개선하는 핵심 전처리 기술입니다.

작동 원리: 전통적 VAD는 에너지(루트 평균 제곱), 스펙트럴 신호 대 잡음비(SNR) 또는 통계적 모델(예: Sohn et al., 1999)의 가설 검정을 사용합니다. WebRTC의 VAD(오픈소스)는 프레임(10/20/30ms) 단위 특징(에너지, ZCR, 스펙트럼)에 기반한 규칙/통계적 결정 로직을 쓰고, 상용·연구 시스템에선 DNN/LSTM 기반의 확률적 VAD가 잡음에 더 강합니다.
실무적 고려: 프레임 크기, 리샘플링(16kHz 권장), 공격성(aggressiveness) 설정, 후처리(연속성 유지용 hangover/smoothing)와 임계치 튜닝이 중요합니다. 프로덕션에서는 VAD를 웨이크워드/엔드포인트 탐지와 결합해 네트워크 전송량과 ASR 호출 비용을 줄입니다.

활용

실제 사용 사례:
- 사례: WebRTC VAD (Chromium / webrtcvad Python wrapper)
코드:
import webrtcvad, wave
v=webrtcvad.Vad(2) # aggressiveness 0-3
# 16kHz, 16bit mono, 30ms frames

사용처: WebRTC 기반 통화(Chrome, Signal 등)에서 실시간 전송 최적화.

- 사례: Kaldi (음성인식 파이프라인)
명령:
# Kaldi feature -> VAD
compute-vad --energy-delta=5.0 scp:feats.scp ark,scp:vad.ark,vad.scp

사용처: 대규모 ASR 트레이닝/디코딩(예: Kaldi v5.x 기반 프로덕션).

- 사례: Silero VAD, pyannote.audio (DNN 기반)
코드:
# Silero VAD (PyTorch)
from silero_vad import get_speech_timestamps
speech_ts = get_speech_timestamps(audio, model)

사용처: 챗봇/녹음 자동 분할, 오프라인 배치 전처리.

실무 패턴: 10–30ms 프레임, 리샘플 16kHz, 후처리(hangover 200–500ms), VAD+endpointing 분리, DNN 모델은 라이트웨이트(quantize, prune)로 엣지 배포.

Content

Voice Activity Detection (VAD) finds speech vs non-speech regions in audio to reduce bandwidth, lower ASR cost, and improve recognition accuracy in real systems.

How it works: Classic VAD uses frame-level energy, zero-crossing rate, or statistical tests (Sohn et al., 1999). WebRTC's VAD (open-source) combines spectral features and decision logic for 10/20/30ms frames. Modern production systems often use DNN/LSTM/CNN models trained to predict speech probabilities, which are more robust to noise and reverberation. Key operational knobs are frame size, sampling rate (commonly 16 kHz), aggressiveness/threshold, hangover/smoothing, and post-filtering for endpointing.
Engineering trade-offs: Simple energy VAD is cheap and works in clean conditions; neural VADs cost more but reduce false positives in noisy/telephony conditions. In real-time systems, implement frame buffering, lookahead for low-latency smoothing, and quantized models for edge deployment.

Applications

Real-world use cases:
- Case: WebRTC VAD (Chromium / webrtcvad Python wrapper)
Code:
import webrtcvad
v=webrtcvad.Vad(2) # aggressiveness 0-3
# feed 10/20/30ms 16kHz 16-bit mono frames

Where used: Real-time calls (Chrome, Signal) to avoid transmitting silence.

- Case: Kaldi (ASR pipelines)
Command:
# compute VAD from features (Kaldi featbin)
compute-vad --energy-delta=5.0 scp:feats.scp ark:vad.ark

Where used: Large-scale ASR training/decoding (Kaldi v5.x deployments).

- Case: Silero VAD / pyannote.audio (DNN)
Code:
from silero_vad import get_speech_timestamps
speech_ts = get_speech_timestamps(audio, model)

Where used: voicemail segmentation, batching ASR jobs, wake-word endpointing.

Production patterns: use 10–30ms frames, resample to 16kHz, apply hangover/smoothing (200–500ms), separate VAD and endpointing, and quantize/prune DNNs for edge.
